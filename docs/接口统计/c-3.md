## 二、B端接口 — 币种配置管理

> 网关入口：gate-back
> URL前缀：`/admin/api/wallet/currency/...`
> 调用方：B端管理后台
> IDL文件：currency.thrift

### 接口列表

```
B1  POST   /admin/api/wallet/currency/page       币种分页列表      【必须】
B2  POST   /admin/api/wallet/currency/create      创建币种          【必须】
B3  POST   /admin/api/wallet/currency/update      编辑币种          【必须】
B4  POST   /admin/api/wallet/currency/toggle      启禁用币种        【必须】
B5  POST   /admin/api/wallet/currency/setBase     设置基准币种      【必须】
B6  POST   /admin/api/wallet/currency/detail      币种详情          【必须】
B7  POST   /admin/api/wallet/currency/rateLog     汇率日志分页      【必须】
```

### 逐个说明

| 编号 | 接口 | 作用 | 分级 | 说明 |
|------|------|------|------|------|
| B1 | PageCurrency | 分页查询所有币种配置 | 必须 | 支持按币种代码、名称、类型、状态筛选 |
| B2 | CreateCurrency | 创建新币种 | 必须 | 包含代码/名称/类型/符号/精度/千分位/汇率阈值/图标等完整字段 |
| B3 | UpdateCurrency | 编辑已有币种 | 必须 | 所有可编辑字段的修改 |
| B4 | ToggleCurrencyStatus | 启用/禁用切换 | 必须 | 控制该币种是否对C端可见 |
| B5 | SetBaseCurrency | 设置某币种为基准币种 | 必须 | 全局唯一基准币种（USDT），用于汇率换算基准 |
| B6 | DetailCurrency | 查看单个币种完整配置 | 必须 | 包含全部20+管理字段 |
| B7 | PageRateLog | 汇率变更历史日志 | 必须 | 审计追溯：谁在何时改了什么汇率、变动幅度多少 |

**小结**：7个接口全部为"必须"，这是币种配置模块的基础CRUD + 日志，缺一不可。

## 三、C端接口 — 钱包核心

> 网关入口：gate-font
> URL前缀：`/api/wallet/...`
> 调用方：C端用户APP/H5
> IDL文件：currency.thrift + wallet.thrift

### 接口列表

```
W1  POST   /api/wallet/activeCurrency            可用币种列表      【必须】
W2  POST   /api/wallet/exchangeRate               获取汇率          【必须】
W3  POST   /api/wallet/overview                   钱包余额概览      【必须】
```

### 逐个说明

| 编号 | 接口 | 作用 | 分级 | 说明 |
|------|------|------|------|------|
| W1 | ListActiveCurrency | 获取所有已启用币种 | 必须 | 币种切换下拉列表数据源，返回简要字段（代码/名称/符号/图标） |
| W2 | GetExchangeRate | 查询两种币种间汇率 | 必须 | 充值/兑换页面展示"1 BSB = ₫2,620"等参考汇率 |
| W3 | GetWalletOverview | 钱包首页余额概览 | 必须 | 当前币种下的总余额、总冻结、各子钱包(中心/奖励/主播)明细 |

**小结**：3个接口全部"必须"，钱包首页的基本数据。

> 网关入口：gate-font
> URL前缀：`/api/wallet/deposit/...`
> 调用方：C端用户

### 接口列表

```
D1  POST   /api/wallet/deposit/config            充值配置          【必须】
D2  POST   /api/wallet/deposit/create             创建充值订单      【必须】
```

### 逐个说明

|------|------|------|------|------|

## 五、C端接口 — 兑换

> 网关入口：gate-font
> URL前缀：`/api/wallet/exchange/...`
> 调用方：C端用户
> IDL文件：exchange.thrift

### 接口列表

```
E1  POST   /api/wallet/exchange/preview           兑换预览          【必须】
E2  POST   /api/wallet/exchange/create             执行兑换          【必须】
```

### 逐个说明

| 编号 | 接口 | 作用 | 分级 | 说明 |
|------|------|------|------|------|
| E1 | PreviewExchange | 兑换试算预览 | 必须 | 输入源币种金额，展示目标币种到账金额、汇率、赠送金额（如有）；不产生实际操作 |
| E2 | CreateExchange | 执行兑换 | 必须 | 一个事务内完成：扣源币种余额 + 加目标币种中心钱包 + 加奖励钱包（赠送部分） |

**小结**：2个接口全部"必须"。兑换是唯一零外部RPC依赖的业务场景（纯内部操作），适合最先跑通验证。

## 六、C端接口 — 提现

> 网关入口：gate-font
> URL前缀：`/api/wallet/withdraw/...`
> 调用方：C端用户
> IDL文件：withdraw.thrift

### 接口列表

```
T1  POST   /api/wallet/withdraw/config            提现配置          【必须】
T2  POST   /api/wallet/withdraw/account            查询提现账户      【必须】
T3  POST   /api/wallet/withdraw/account/save       保存提现账户      【必须】
T4  POST   /api/wallet/withdraw/create             创建提现订单      【必须】
```

### 逐个说明

| 编号 | 接口 | 作用 | 分级 | 说明 |
|------|------|------|------|------|
| T1 | GetWithdrawConfig | 提现页面配置 | 必须 | 可用提现方式(银行卡/电子钱包/USDT) + 限额 + 费率 + 到账时间；需联合KYC状态过滤渠道 |
| T2 | GetWithdrawAccount | 查询已保存的提现账户 | 必须 | 返回用户上次使用的银行卡/电子钱包信息，免重复输入 |
| T3 | SaveWithdrawAccount | 保存/更新提现账户 | 必须 | 首次提现时填写银行名/卡号/户名等信息，保存后下次复用 |
| T4 | CreateWithdrawOrder | 创建提现订单 | 必须 | 核心操作：校验限额 → 冻结余额 → RPC提交财务审核 → 返回状态 |

**小结**：4个接口全部"必须"。提现是依赖链最长的业务场景（涉及KYC + 财务审核 + 冻结/解冻/扣除三件套）。

## 八、RPC暴露接口 — 供其他模块调用

> 协议：Kitex RPC（不经过HTTP网关）
> 调用方：ser-finance（当前唯一确定调用方）
> IDL文件：wallet_rpc.thrift
> 详细分析：/Users/mac/gitlab/z-readme/rpc暴露/c-3.md

### 接口列表

```
RPC-E1  WalletCredit           充值成功入账（中心钱包）      【必须】
RPC-E2  WalletCreditReward     奖金入账（奖励钱包）          【必须】
RPC-E3  WalletFreeze           冻结可用余额                  【必须】
RPC-E4  WalletUnfreeze         解冻归还冻结余额              【必须】
RPC-E5  WalletDeduct           最终扣除冻结资金              【必须】
RPC-E6  WalletManualAdd        人工加款（多子钱包）          【必须】
RPC-E7  WalletManualSub        人工减款（多子钱包）          【必须】
RPC-E8  GetUserBalance         查询用户各子钱包余额          【必须】
RPC-E9  GetCurrencyConfig      获取币种配置信息              【必须】
```

### 逐个说明

| 编号 | 接口 | 触发场景 | 分级 | 关键设计点 |
|------|------|---------|------|----------|
| RPC-E1 | WalletCredit | 财务确认充值成功/补单 → 入账到中心钱包 | 必须 | 幂等key=orderNo，同一充值单只入一次 |
| RPC-E2 | WalletCreditReward | 充值赠送/活动奖金 → 入账到奖励钱包 | 必须 | 附带turnoverMulti流水倍数要求 |
| RPC-E3 | WalletFreeze | 用户发起提现 → 冻结对应金额 | 必须 | available -= amount, frozen += amount |
| RPC-E4 | WalletUnfreeze | 审核驳回/出款失败 → 归还冻结 | 必须 | 只需orderNo即可定位冻结记录 |
| RPC-E5 | WalletDeduct | 出款成功 → 真正扣除冻结资金 | 必须 | 与Unfreeze互斥终态 |
| RPC-E6 | WalletManualAdd | B端人工加款审核通过 → 多子钱包一次性加款 | 必须 | 一个事务内最多4个子钱包同时加 |
| RPC-E7 | WalletManualSub | B端人工减款审核通过 → 多子钱包分别减 | 必须 | 余额不足时实际扣减=当前余额（扣到0） |
| RPC-E8 | GetUserBalance | 人工修正弹窗/风控查看 → 展示余额 | 必须 | 纯查询，无幂等要求 |
| RPC-E9 | GetCurrencyConfig | 统计报表/金额格式化 → 获取精度和汇率 | 必须 | 建议调用方做本地缓存 |

### 核心设计约束

- **全部金额用 string 类型**（ADR-06：传输string + 运算shopspring/decimal + 存储DECIMAL(30,8)）
- **资金操作全部幂等**：基于 `orderNo + flow_type` 唯一索引，重复调用返回成功不重复执行
- **冻结三件套互斥**：Freeze → 只能走 Unfreeze（还钱）或 Deduct（扣钱），二选一终态

**小结**：9个RPC全部"必须"。这是财务模块的命脉——没有这些，充值入账、提现审核、人工修正全部无法完成。

### 开发阶段Mock策略

| 接口 | 是调料还是火 | Mock方案 |
|------|-------------|---------|
| RPC-C1~C4 | **调料**（可Mock继续开发） | 硬编码返回充值/提现方式列表、限额配置，主流程先跑通 |
| RPC-C5 | **调料** | 固定返回 status=3（已认证），全渠道可用 |
| RPC-C6~C8 | **调料** | 已存在可直接对接，或后期补上 |
| 币种配置内部方法 | **火** | 同进程调用，必须先实现，否则钱包业务层完全无法工作 |

**小结**：核心阻塞点是 ser-finance 的4个RPC（P0），IDL定义需要尽早与财务团队对齐。其余4个已存在模块，可直接对接。

## 十、后台任务与回调

> 非接口形式，但同属ser-wallet需要实现的职责

### 任务列表

```
BG1  定时任务  汇率自动同步        拉取三方汇率API，按阈值判断是否更新     【必须】
BG2  定时任务  充值订单超时处理    超过N分钟未支付的充值订单自动关闭        【应该】
BG3  回调     三方支付通道回调     接收充值/提现状态通知（待确认由谁接收）  【商榷】
```

| 编号 | 类型 | 作用 | 分级 | 说明 |
|------|------|------|------|------|
| BG1 | 定时任务 | 汇率自动同步 | 必须 | 币种配置模块职责；定时拉取外部汇率API，与当前汇率对比，超过阈值则更新并写日志 |
| BG2 | 定时任务 | 充值订单超时 | 应该 | 避免大量"待支付"订单永久挂着；超时后标记为失败 |
| BG3 | 回调接口 | 三方通道回调 | 商榷 | 充值回调由谁接收（钱包还是财务）尚未确定；这决定了钱包是否需要直接对接三方SDK |
