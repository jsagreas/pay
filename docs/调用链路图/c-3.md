## 一、请求入口总览

任何请求进入 ser-wallet 只有**三扇门**：

```
外部世界
    │
    ├── 门1：gate-font（C端用户请求）
    │   │  POST /api/wallet/...
    │   │  携带：Token → 中间件解析出 userId
    │   │  特点：高并发、面向用户、要快
    │   ▼
    │   ser-wallet（14个C端方法）
    │
    ├── 门2：gate-back（B端管理员请求）
    │   │  POST /admin/api/wallet/...
    │   │  携带：AdminToken → 中间件解析出 adminId + 权限
    │   │  特点：低并发、面向管理员、要安全
    │   ▼
    │   ser-wallet（7个B端方法）
    │
    └── 门3：Kitex RPC（内部模块调用）
        │  rpc.WalletClient().XxxMethod(ctx, req)
        │  携带：内部服务身份
        │  特点：模块间调用、幂等要求高
        ▼
        ser-wallet（9个RPC方法）
```

**一个请求的通用骨架**：

```
请求到达 → 网关鉴权 → 路由到Handler → 参数校验 → Service业务逻辑
    → [可能]查币种配置（同进程）
    → [可能]查/改余额（DB事务）
    → [可能]调外部RPC（跨模块）
    → [可能]写流水记录（不可变）
    → 组装响应 → 返回
```

下面逐条链路展开，**每一跳都标清楚**。

## 二、链路1：用户查看钱包首页 ★

**触发**：用户打开APP钱包页面
**涉及接口**：`ListActiveCurrency` + `GetWalletOverview`
**跨模块**：无
**复杂度**：★（纯查询，最简单的链路）

```
时序图：

用户APP          gate-font         ser-wallet              DB(db_wallet)
  │                 │                  │                       │
  │  打开钱包页面    │                  │                       │
  ├────────────────►│                  │                       │
  │                 │  ① POST /api/wallet/activeCurrency       │
  │                 ├─────────────────►│                       │
  │                 │                  │  [必须]查 t_currency   │
  │                 │                  │  WHERE status=1        │
  │                 │                  │  ORDER BY sort_value   │
  │                 │                  ├──────────────────────►│
  │                 │                  │◄──────────────────────┤
  │                 │                  │  [流程]组装简要字段     │
  │                 │                  │  (code/name/symbol/icon)│
  │                 │◄─────────────────┤                       │
  │◄────────────────┤  返回币种列表     │                       │
  │                 │                  │                       │
  │  选中某个币种    │                  │                       │
  │  (如 VND)       │                  │                       │
  ├────────────────►│                  │                       │
  │                 │  ② POST /api/wallet/overview              │
  │                 │     body: {currencyCode: "VND"}           │
  │                 ├─────────────────►│                       │
  │                 │                  │  [必须]查 t_wallet_account │
  │                 │                  │  WHERE user_id=X       │
  │                 │                  │    AND currency_code="VND" │
  │                 │                  ├──────────────────────►│
  │                 │                  │◄──────────────────────┤
  │                 │                  │                       │
  │                 │                  │  [判断]账户是否存在？   │
  │                 │                  │  ├─ 不存在 → 惰性初始化 │
  │                 │                  │  │  INSERT 3个子钱包    │
  │                 │                  │  │  (中心/奖励/主播)    │
  │                 │                  │  │  余额均为0           │
  │                 │                  │  └─ 已存在 → 直接读取   │
  │                 │                  │                       │
  │                 │                  │  [必须]查 t_currency    │
  │                 │                  │  获取精度/符号/千分位   │
  │                 │                  │  (同进程内调用,无RPC)   │
  │                 │                  │                       │
  │                 │                  │  [流程]格式化金额       │
  │                 │                  │  按precision/千分位格式化│
  │                 │◄─────────────────┤                       │
  │◄────────────────┤                  │                       │
  │                 │  返回余额概览     │                       │
  │                 │  {totalBalance, totalFrozen, subWallets[]}│
  │                 │                  │                       │
```

**这条链路的要点**：
- 零外部RPC依赖
- 惰性初始化：用户首次访问某币种时自动创建3个子钱包
- 金额格式化依赖币种配置（精度/千分位/小数点）
- 缓存策略：币种配置可进程内缓存（sync.Map），5分钟TTL

## 三、链路2：B端币种配置管理 ★

**触发**：管理员在后台操作币种
**涉及接口**：`CreateCurrency` / `UpdateCurrency` / `ToggleCurrencyStatus` / `SetBaseCurrency`
**跨模块**：ser-blog（审计日志，oneway异步）、ser-s3（图标上传）
**复杂度**：★（标准CRUD）

以"创建币种"为例：

**"设置基准币种"的特殊判断**：
```
[判断] 是否已有基准币种？
  ├─ 已有 → 拒绝（基准币种只能设一次，不可变更）
  └─ 没有 → 将该币种 is_base=1
             [判断] 该币种是否已启用？
               ├─ 未启用 → 自动启用（基准币种不可禁用）
               └─ 已启用 → 正常设置
```

## 四、链路3：汇率自动同步 ★★

**触发**：进程内定时任务（每N分钟执行一次）
**涉及接口**：无外部接口，内部定时任务
**跨模块**：三方汇率API（HTTP外部调用）
**复杂度**：★★（涉及分布式锁、阈值判断、多币种循环）

```
                ser-wallet进程                    Redis         三方汇率API        DB
                    │                              │               │              │
                    │ [循环]cron定时触发(每5min)     │               │              │
                    │                              │               │              │
                    │ [必须]获取分布式锁             │               │              │
                    ├─────────────────────────────►│               │              │
                    │  SETNX rate_sync_lock TTL=60s │               │              │
                    │◄─────────────────────────────┤               │              │
                    │                              │               │              │
                    │ [判断]拿到锁了吗？             │               │              │
                    │  ├─ 没拿到 → 跳过本轮（其他实例在执行）         │              │
                    │  └─ 拿到了 → 继续             │               │              │
                    │                              │               │              │
                    │ [必须]查所有已启用币种          │               │              │
                    ├──────────────────────────────────────────────────────────►│
                    │  SELECT * FROM t_currency WHERE status=1                  │
                    │◄──────────────────────────────────────────────────────────┤
                    │                              │               │              │
                    │ [循环]遍历每个非基准币种        │               │              │
                    │  ┌──────────────────────────────────────────────────────┐ │
                    │  │ for currency in currencies (排除USDT基准)：          │ │
                    │  │                            │               │        │ │
                    │  │ [必须]调三方API获取实时汇率   │               │        │ │
                    │  ├──────────────────────────────────────────►│        │ │
                    │  │  GET /api/rate?from=VND&to=USDT            │        │ │
                    │  │◄──────────────────────────────────────────┤        │ │
                    │  │  real_rate = 三方返回值       │               │        │ │
                    │  │                            │               │        │ │
                    │  │ [挑战]三方API不可用？         │               │        │ │
                    │  │  ├─ 不可用 → 跳过该币种，记录告警日志       │        │ │
                    │  │  └─ 可用 → 继续             │               │        │ │
                    │  │                            │               │        │ │
                    │  │ [判断]计算偏差               │               │        │ │
                    │  │  deviation = |real - platform| / real × 100%│        │ │
                    │  │  ├─ deviation < threshold% → 不更新，跳过   │        │ │
                    │  │  └─ deviation ≥ threshold% → 需要更新       │        │ │
                    │  │                            │               │        │ │
                    │  │ [必须]更新平台汇率            │               │        │ │
                    │  │  new_platform = real_rate   │               │        │ │
                    │  │  new_deposit = platform × (1 + fluctuation%)│        │ │
                    │  │  new_withdraw = platform × (1 - fluctuation%)       │ │
                    │  │                            │               │        │ │
                    │  │ [必须]UPDATE t_currency     │               │        │ │
                    │  ├──────────────────────────────────────────────────►│ │
                    │  │◄──────────────────────────────────────────────────┤ │
                    │  │                            │               │        │ │
                    │  │ [必须]INSERT t_rate_log (审计不可变)         │        │ │
                    │  ├──────────────────────────────────────────────────►│ │
                    │  │◄──────────────────────────────────────────────────┤ │
                    │  │                            │               │        │ │
                    │  │ [流程]更新进程内缓存          │               │        │ │
                    │  │ [流程]更新Redis缓存          │               │        │ │
                    │  ├─────────────────────────────►│               │        │ │
                    │  │                            │               │        │ │
                    │  └──────────────────────────── 下一个币种 ──────────────┘ │
                    │                              │               │              │
                    │ [必须]释放分布式锁             │               │              │
                    ├─────────────────────────────►│               │              │
                    │  DEL rate_sync_lock           │               │              │
```

**挑战标注**：
- [挑战] 多实例部署时只能一个执行 → Redis SETNX分布式锁
- [挑战] 三方API不可用 → 跳过本轮，保留上次汇率，不能让系统停摆
- [挑战] 汇率更新时正好有用户在创建订单 → 锁汇快照机制解决（订单创建时获取当时汇率并写入订单）

**触发**：C端用户发起充值

## 十一、全景时序总图

**一个用户从注册到充值到兑换到提现的完整生命周期**：

```
时间轴 ──────────────────────────────────────────────────────────────────────►

1. 首次打开钱包
   用户APP → gate-font → ser-wallet
   [惰性初始化] 创建 VND×3子钱包 (中心/奖励/主播，余额均为0)
   耗时: ~50ms | 跨模块: 无 | 复杂度: ★

2. 充值 ₫1,000,000（银行转账）
   用户APP → gate-font → ser-wallet → ser-finance → 三方通道
   [创建订单] → [跳转支付页] → [用户转账] → [三方回调]
   → ser-finance → ser-wallet.WalletCredit → 中心钱包+₫1,000,000
   耗时: ~5-30min(含用户操作) | 跨模块: 3 | 复杂度: ★★★★

3. 兑换 ₫500,000 → BSB
   用户APP → gate-font → ser-wallet
   [预览] → [确认] → [一个事务: 扣VND+加BSB中心+加BSB奖励]
   中心钱包 VND: 1,000,000→500,000 | BSB中心: 0→190.84 | BSB奖励: 0→19.08
   耗时: ~200ms | 跨模块: 无 | 复杂度: ★★

4. 提现 ₫300,000（银行卡）
   用户APP → gate-font → ser-wallet → ser-kyc → ser-finance → 三方代付
   [KYC校验] → [获取配置] → [冻结₫300,000] → [提交审核]
   → [风控通过] → [财务通过] → [代付出款] → [通道回调]
```
