### 2.4 统一响应结构

来源：`common/pkg/ret/project_ret.go`

**HTTP响应体**：
```go
type R struct {
    TraceId string         `json:"traceId"`  // 链路追踪ID
    Code    errs.ErrorType `json:"code"`     // 0=成功，1=严重错误，其他=业务错误码
    Msg     string         `json:"msg"`      // 成功时为空，错误时为i18n翻译后的提示语
    Data    interface{}    `json:"data"`     // 业务数据
}
```

**RPC侧（Kitex BizStatusError）**：
```go
// 构造业务错误（在service层使用）
ret.BizErr(ctx, errs.SomeCode)                    // 自动从Redis i18n哈希查找msg
ret.BizErr(ctx, errs.SomeCode, "自定义消息")      // 指定msg

// 包装RPC响应（在service层使用，零值检查）
ret.RpcOne[T](ctx, result, err, errs.DefaultCode) // 如果err!=nil则转为BizErr

// 提取错误码
ret.GetBizErrCode(err, defaultCode)               // 从BizStatusError中提取code
```

**HTTP侧（在网关Handler中使用）**：
```go
// 成功响应
ret.HttpOk(ctx, data)           // → (200, R{Code:0, Data:data})

// 错误响应
ret.HttpErr(ctx, err, i18nClient)  // → (200, R{Code:bizCode, Msg:翻译后消息})

// 组合便捷方法（最常用）
ret.HttpOne(ctx, data, rpc.I18nClient(), err)  // err!=nil → HttpErr, 否则 → HttpOk
```

**i18n消息查找机制**：
- Redis哈希键：`i18n:hs:zhcn`
- 字段名：错误码数字字符串（如 `"6011001"`）
- 查找失败时降级为 `bizErr.BizMessage()`（Kitex内置的错误消息）

## 第三部分：ETCD客户端

来源：`common/pkg/etcd/`

### 3.1 初始化

```go
// 从环境变量读取ETCD地址
// 环境变量：ETCD_ADDR=192.168.27.61:2379,192.168.27.62:2379,192.168.27.63:2379
etcd.InitEtcd()        // 仅初始化连接（sync.Once幂等）
etcd.Address() []string // 返回ETCD端点列表
```

### 3.2 缓存Watch模式（主要使用方式）

```go
// 加载前缀下所有键值到本地缓存，并持续Watch变更
etcd.LoadAndWatch("/slg/conf/")    // 全局配置Watch
etcd.LoadAndWatch("/slg/")         // 更宽泛的Watch（含/slg/serv/、/slg/switch/等）

// 从缓存读取（零网络开销，lock-free atomic.Value）
etcd.Get(ctx, "/slg/conf/tidb/addr")           // 返回string，缓存未命中返回""
etcd.GetDef(ctx, "/slg/conf/tidb/addr", "默认值") // 带默认值

// 注册键值变更回调（实时监听配置热更新）
etcd.Hook("/slg/switch/xxx", func(val string) {
    // 配置变更时自动触发
})
```

**自定义Watcher**（独立前缀独立缓存）：
```go
w := etcd.CustomWatcher("/slg/switch/wallet/")
w.Get(ctx, key)
w.Hook(key, callback)
```

### 3.3 直读直写模式（绕过缓存）

```go
etcd.DirectGet(ctx, key)                          // 直接读ETCD（用于初始化时获取port/db等）
etcd.DirectGetListByPrefix(ctx, prefix)           // 前缀批量读
etcd.DirectPut(ctx, key, val)                     // 直接写
etcd.DirectDelete(ctx, key)                       // 直接删
etcd.DirectPutWithLease(ctx, key, val, ttlSecs)   // 带TTL的写入（自动过期）
```

**使用场景区分**：
- `DirectGet` → 服务启动时获取 port、db名等一次性配置
- `Get`/`GetDef` → 运行时读取频繁访问的配置（性能高，零网络开销）
- `Hook` → 配置热更新场景（如开关切换、汇率参数变更）

### 5.2 UserPlugin 审计插件（自动填充审计字段）

来源：`common/pkg/db/mysql/user_plugin.go`

**自动填充的字段**：
| 字段名 | 类型 | CREATE时 | UPDATE时 | 值来源 |
|--------|------|---------|---------|-------|
| CreateBy | int64 | 填充 | — | `tracer.GetUserId(db.Statement.Context)` |
| CreateAt | int64 | 填充 | — | `time.Now().UnixMilli()` |
| UpdateBy | int64 | 填充 | 填充 | `tracer.GetUserId(db.Statement.Context)` |
| UpdateAt | int64 | 填充 | 填充 | `time.Now().UnixMilli()` |

**工作机制**：
- 通过GORM Callback注册：`Before("gorm:create")` 和 `Before("gorm:update")`
- 反射查找字段名，支持值类型和指针类型
- 支持单条和批量（slice/array）操作
- userId为0时（如系统任务）不填充 CreateBy/UpdateBy

**对 ser-wallet 的意义**：
- 所有表都应包含 `create_by`、`create_at`、`update_by`、`update_at` 字段
- 不需要在service层手动填充这些字段，Plugin自动处理
- 前提：`ctx` 必须正确传递（从tracer中提取userId）

来源：`common/pkg/db/cmd/repo_template.go`

每个表自动生成以下free function（无接收器，包级函数）：

```go

UpdateByConds(ctx, entity, conds...)   error           // 按条件更新

UpdateFieldsByConds(ctx, entity, conds, fields...) error

// 删除
DeleteById(ctx, ids...)                error           // 硬删除
DeleteByConds(ctx, conds...)           error           // 硬删除

// 查询
GetOneById(ctx, id)                    (*Model, error) // 未找到返回nil不报错
GetOneByConds(ctx, conds...)           (*Model, error) // 未找到返回nil
GetListByIds(ctx, ids)                 ([]*Model, error)
GetListByConds(ctx, conds...)          ([]*Model, error) // 上限10000条
GetPageByConds(ctx, pageNo, pageSize, orderby, conds...) (*dto.PageResult[Model], error)
GetValuesByConds[T](ctx, field, conds...) ([]T, error) // 单字段pluck

```

**分页结果结构**：
```go
type PageResult[T any] struct {

    Total     int64 `json:"total"`
    PageTotal int64 `json:"page_total"`
    PageNo    int   `json:"page_no"`
    PageSize  int   `json:"page_size"`
}
```

**关键注意点**：

### 6.4 三个通用Handler

来源：`common/rpc/handler.go`

**Handler[REQ, RESP]** — 标准RPC调用（最常用）：
```go
func Handler[REQ any, RESP any](
    ctx context.Context,
    c *app.RequestContext,
    rpcMethod func(context.Context, REQ, ...callopt.Option) (RESP, error),
)
```
工作流程：`BindAndValidate(请求体) → rpcMethod(ctx, req) → c.JSON(ret.HttpOne(ctx, resp, I18nClient(), err))`

**HandlerNoParam[RESP]** — 无参数RPC调用：
```go
func HandlerNoParam[RESP any](
    ctx context.Context,
    c *app.RequestContext,
    rpcMethod func(context.Context, ...callopt.Option) (RESP, error),
)
```
工作流程：`rpcMethod(ctx) → c.JSON(ret.HttpOne(ctx, resp, I18nClient(), err))`

**HandlerWithCSV[REQ, RESP]** — CSV下载RPC调用：
```go
func HandlerWithCSV[REQ any, RESP any](...)
```
工作流程：与Handler相同，但如果RESP实现了 `CSVResponse` 接口（`GetFilename()` + `GetData()`），则返回文件流而非JSON。

### 7.2 gen_kitex.sh 代码生成脚本

| Api前缀 | `/admin/api` | `/api` |
| IP策略 | **白名单**（fail-closed：出错拒绝） | **黑名单**（fail-open：出错放行） |
| IP中间件 | `WhiteCheck()` → `IpClient().OnWhiteList()` | `BlackCheck()` → `IpClient().OnBlackList()` |
| 认证中间件 | `BackAuthMiddleware()` | `FontAuthMiddleware()` |
| Token Redis键 | token本身（无前缀） | `"user:token:" + token`（有前缀） |
| Session类型 | `BUserTokenPayload`（ID + UserName） | `CUserTokenPayload`（ID + NickName） |
| RBAC | **有** — `BUserClient().CheckUrlAuth(ctx, fullPath)` | **无** — Token有效即可 |
| Body上限 | 20MB | 默认 |

### 8.4 中间件链详解

**SetHeaderInfo()**（两个网关共用，`middleware/info.go`）：
```
提取并注入到metainfo上下文：
  ClientIP     ← c.ClientIP()
  UserLang     ← X-User-Language header
  DeviceID     ← X-User-Device-ID header
  DeviceType   ← X-User-Device-Type header
  UserToken    ← Authorization header → 降级到 session_token cookie
```

**BackAuthMiddleware()**（gate-back `/admin/api`）：
```
1. 从metainfo获取token → 为空则拒绝（TokenNotExist）
2. Redis直接GET(token) → 为空则拒绝（TokenAnalysisFailed）
3. 反序列化为 BUserTokenPayload → ID为0则拒绝
4. 注入 UserIdHeader + UserNameHeader 到metainfo
5. 调用 rpc.BUserClient().CheckUrlAuth(ctx, c.FullPath()) → 未授权则拒绝
```

**FontAuthMiddleware()**（gate-font `/api`）：
```
1. 从metainfo获取token → 为空则拒绝
2. Redis GET("user:token:" + token) → 为空则拒绝
3. 反序列化为 CUserTokenPayload → ID为0则拒绝
4. 注入 UserIdHeader + UserNameHeader(NickName) 到metainfo
5. 无RBAC校验，直接放行
```

### 9.6 Rep(Repository)层编码模式

**两层Repo结构**：

| 层级 | 路径 | 性质 | 修改方式 |
|------|------|------|---------|
| 自动生成 | `gorm_gen/repo/{table}r/{table}r.go` | 20个标准CRUD | **禁止手动修改**，由gorm_gen.go重新生成 |
| 手写业务 | `rep/{entity}.go` | 复杂查询/事务 | 开发者自行编写 |

**手写Repo模式**：

```go
type WalletRepo struct{}

func NewWalletRepo() *WalletRepo {
    cfg.InitMySQL()  // 确保MySQL已初始化（懒加载，幂等）
    return &WalletRepo{}
}

// 使用query.Q（GORM Gen生成的类型安全查询）
func (r *WalletRepo) GetByUserAndType(ctx context.Context, userId int64, walletType int32) (*model.Wallet, error) {
    w := query.Q.Wallet
    return w.WithContext(ctx).
        Where(w.UserID.Eq(userId)).
        Where(w.WalletType.Eq(walletType)).
        Where(w.DeleteAt.Eq(0)).   // 软删除条件
        First()
}

// 使用事务
func (r *WalletRepo) TransferInTx(ctx context.Context, from, to *model.Wallet, amount int64) error {
    return query.Q.Transaction(func(tx *query.Query) error {
        // 扣减
        _, err := tx.Wallet.WithContext(ctx).
            Where(tx.Wallet.ID.Eq(from.ID)).
            UpdateSimple(tx.Wallet.Balance.Sub(amount))
        if err != nil { return err }
        // 增加
        _, err = tx.Wallet.WithContext(ctx).
            Where(tx.Wallet.ID.Eq(to.ID)).
            UpdateSimple(tx.Wallet.Balance.Add(amount))
        return err
    })
}

// 使用map进行部分更新（避免零值问题）
func (r *WalletRepo) UpdatePartial(ctx context.Context, id int64, updates map[string]interface{}) error {
    w := query.Q.Wallet
    _, err := w.WithContext(ctx).Where(w.ID.Eq(id)).Updates(updates)
    return err
}
```

**软删除约定**：
- `delete_at = 0` 表示未删除
- `delete_at = 非零时间戳` 表示已删除（值为删除时的UnixMilli）
- 所有查询默认需要加 `.Where(w.DeleteAt.Eq(0))` 条件
- 删除操作使用 `UpdateSimple(w.DeleteAt.Value(time.Now().UnixMilli()))` 而非 `Delete`

### 13.3 关键易错点

1. **MySQL懒加载时序**：`query.SetDefault(db)` 必须在任何GORM Gen查询之前执行，否则 `query.Q` 为nil导致panic
2. **etcd.LoadAndWatch vs etcd.InitEtcd**：如果main()中需要用 `etcd.Get()`（非Direct），必须先 `LoadAndWatch`
3. **handler.go不要写逻辑**：任何验证、转换、业务判断都放在 internal/ser/ 中
4. **Update零值问题**：自动生成的 `Update()` 忽略零值 → 需要设为0/""时用 `UpdateFields` 或 `map[string]interface{}`
5. **硬删除vs软删除**：自动生成的 `DeleteById`/`DeleteByConds` 是硬删除 → 软删除需手动UpdateSimple设置delete_at
6. **所有查询加软删除条件**：`.Where(w.DeleteAt.Eq(0))`
7. **RBAC路由注册**：gate-back的 `/admin/api/` 路由必须在菜单权限表中配置
8. **RPC超时3秒**：跨服务调用默认3秒超时，复杂链路注意超时级联
9. **Redis键前缀**：C端token是 `"user:token:" + token`，B端token直接是 `token`
10. **metainfo透传**：tracer注入的Header会跨RPC自动透传，不需要手动传递userId等
