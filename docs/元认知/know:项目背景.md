# 背景说明
## 零、项目背景
- 公司决定从0到1开发一个面向多地区多功能的基于golang语言开发的WEB应用【也就是泛娱乐+合规竞彩平台产品；B/C 双端 + 泛娱乐（社交 / 直播 / 短视频 / 语聊 / 游戏）+ 合规竞彩博彩 + 分布式微服务大型复杂系统】
- 包括但不限于具有如下业务场景：社交，直播，短视频，语聊，游戏，竞彩等，具备B端综合管理运用功能，C端面向客户【包括app,h5,web页面】
- 包括但不限于具有如下业务功能：【业务域包括但不限于：用户、内容、互动、支付、资产、交易、礼物、结算、赛事、投注、赔率、风控、合规、运营、推荐、数据、后台管理等完整业务域体系】
## 一、基础通用域（所有上层业务的地基）
- 用户域（User）注册、登录、注销、封禁、身份、账号体系、账号状态、实名认证、年龄校验、设备绑定、设备指纹、多账号合规校验
- 身份认证与权限域（Auth / IAM / RBAC） 登录态、Token、Session、角色、权限、授权链路、权限点、权限继承、API 鉴权
- 组织与多角色域（Org / Role） 普通用户、主播、房主、代理、运营、风控、管理员、超管等、公会 / 战队 / 代理体系、多层级组织关系、结算主体绑定
- 内容域（Content） 文本、图片、视频、语音、房间内容、UGC/PGC 管理
- 关系域（Social Graph）  关注、粉丝、好友、拉黑、屏蔽、关系链
- 消息与通知域（IM / Notification） 私聊、群聊、系统消息、推送、站内信
- 支付域（Payment） 充值、消费、退款、对账、支付渠道
- 资产 / 钱包域（Wallet / Asset）余额、虚拟币、积分、道具、流水
- 订单 / 交易域（Order / Trade） 订单生命周期、状态流转、幂等、风控前置
- 风控与安全域（Risk / Security）反作弊、反刷、账号风控、支付风控、内容风控
- 配置与参数中心域（Config / Param）运营配置、全局配置、业务参数、动态规则、玩法参数、风控阈值、博彩限额参数
- 功能开关与灰度域（Feature Flag）灰度发布、AB 实验、定向开关、分人群生效、功能开关、人群定向
- 日志、审计与追溯域（Log / Audit）操作日志、审计日志、合规追溯、合规审计日志、投注行为审计
## 二、内容与社交基础域（泛娱乐核心）
- 内容资产域（Content Asset）视频、语音、图片、文本、封面、媒体资源、CDN 资源、媒体转码
- 内容生产与发布域（UGC / PGC）发布、编辑、下架、置顶、推荐权重、内容状态流转、内容标签
- 社交关系域（Social Graph）关注、粉丝、好友、拉黑、屏蔽
- 互动行为域（Interaction） 点赞、评论、回复、弹幕、表情互动
- 消息与即时通讯域（IM / Message）私聊、群聊、系统消息、公告 、房间聊天
- 通知与推送域（Notification）站内信、Push、短信、邮件、系统通知
- 通知与任务调度域（Scheduler / Job）定时任务、延迟任务、异步处理
## 三、实时互动与会话场景域（直播 / 语聊 / 游戏）
- 房间与会话域（Room / Session）房间生命周期 、房间状态、成员管理
- 上下麦与连麦域（Mic / Link）上麦、下麦、连麦、PK
- 实时状态同步域（Presence / State）在线状态、心跳、同步
- 弹幕与实时互动域（Realtime Interaction）弹幕流控、互动节奏控制 、弹幕限流 、实时互动节奏
- 房间 / 会话域（Room / Session）房间生命周期、上下麦、状态管理
- 互动域（Interaction）点赞、评论、礼物、弹幕、连麦
- 玩法 / 规则域（Game / Rule）玩法配置、规则引擎、状态机
- 对局 / 赛事域（Match / Event）房间对局、赛事进程、结果确认
## 四、支付、资产与交易基础域（所有钱相关）
- 支付域（Payment）充值、扣款、退款、支付渠道
- 钱包与资产域（Wallet / Asset）余额、虚拟币、积分、道具 、勋章 、会员权益
- 订单域（Order）订单创建、状态机、幂等控制、普通订单 、投注订单 、礼物订单
- 交易流水与账本域（Transaction / Ledger） 资金流水、账务明细 、虚拟资产流水
- 对账与清结算基础域（Reconciliation）内部对账、渠道对账 、内部账务核对
## 五、礼物、道具与虚拟经济系统
- 礼物与道具配置域（Gift / Props）道具定义、道具配置、礼物配置、消耗规则、展示规则
- 虚拟经济规则域（Economy Rule）价格体系、产出与消耗平衡 、通胀 / 回收
- 分成与收益计算域（Revenue Split）主播分成、公会分成、平台抽成
- 结算域（Settlement） 周期结算、结算单、打款准备
- 礼物与道具域（Gift / Props）礼物配置、消耗规则、分成计算
- 结算 / 分成域（Settlement）主播分成、公会分账、周期结算 、对外打款
## 六、竞彩 / 合规博彩核心域（重点）
- 赛事与赛程域（Sport / Match）联赛、赛事、赛程、状态同步
- 玩法与投注规则域（Bet Rule）投注类型、规则校验、玩法引擎 、玩法定义
- 赔率与盘口域（Odds / Market）赔率生成、盘口调整、封盘
- 投注单域（Bet Order / Ticket） 投注创建、校验、确认、锁单
- 投注风控域（Bet Risk Control）限额、限频、异常投注识别
- 开奖与结果确认域（Result / Settle）比赛结果、结算触发
- 返奖与派奖域（Payout）返奖计算、派奖执行
- 合规与监管域（Compliance）地域限制、用户资格校验、合规报送 、反洗钱报表
- 赔率与策略调控域（Trading / Strategy）风险敞口、动态调盘
- 奖励与结算域（Reward） 奖励发放、返奖、派奖
- 概率与风控策略域（Odds / Strategy）概率控制、赔率、策略调整
## 七、真人视讯与荷官域（Live Casino / Dealer）
- 真人桌台与牌局域（Table/GameRound）桌台管理（百家乐 / 轮盘 / 龙虎等）、局 / 轮 / 回合 生命周期、桌台状态（空闲 / 投注中 / 封盘 / 结算）
- 荷官与真人操作域（DealerOperation） 荷官账号与身份、荷官排班、发牌 / 操作行为记录、行为审计（关键合规点）
- 真人视频流与互动控制域（LiveStreamControl） 真人视频流接入、多路摄像头切换、延迟控制、视频与下注状态同步 、不能简单归为“直播域”，因为它必须与投注、结果强绑定
- 真人下注窗口与封盘控制域（BetWindowControl）、投注开始 / 截止、封盘逻辑、投注有效性校验
- 真人结果确认与公平性域（Fairness&Result）、结果确认、视频回放追溯、争议核查、公平性证明（录像、日志）
## 八、风险、安全与反作弊域（平台级）
- 账号安全与风控域（Account Risk）异常登录、设备指纹、设备风险 、账号盗用、多设备异常、身份冒用
- 内容安全与审核域（Moderation）  机审、人审、举报、封禁、申诉、合规内容审计
- 资金安全与反洗钱域（AML / Anti-Fraud） 异常资金流、洗钱识别、资金通道滥用、监管报送
- 行为反作弊域（Anti-Cheat）  刷量、刷单、机器人、机机器人识别、羊毛党、自动化脚本
- 业务风控与交易风控域（Business / Transaction Risk）、游戏对局行为风控、活动与运营风控、投注与博彩风控、资金交易风控 、规则滥用与奖励套利识别
## 八、推荐、分发与增长域
- 推荐策略域（Recommendation）策略、权重、召回、排序
- 推荐与分发域（Recommend / Feed）信息流、推荐策略、排序、曝光控制
- 信息流与分发域（Feed / Distribution）内容分发、内容曝光、搜索归属、曝光控制
- 榜单与排序域（Ranking）   热度榜、收益榜、赛事榜
- 增长与激励域（Growth） 拉新、留存、裂变
## 十、运营与活动域（B 端核心）
- 活动与任务域（Campaign / Task）活动配置、任务系统
- 运营域（Operation）活动、任务、运营位、弹窗、榜单配置
- 运营位与资源位域（Banner / Slot）  推荐位、广告位
- 运营策略域（Operation Strategy）  人群策略、精细化运营
## 十一、客服、申诉与用户仲裁支持域
- 客服与工单域（Support/ Ticket）客服系统、问题流转、反馈、投诉、工单流转
- 申诉与仲裁域（Appeal / Arbitration）封禁申诉、投注争议
## 十二、数据与平台能力域（技术支撑）
- 埋点与行为采集域（Tracking）行为日志、行为采集、事件日志、事件采集
- 数据分析与报表域（Analytics / BI） 业务分析、运营报表
- 统计与分析域（Analytics）行为埋点、数据报表、转化分析
- 数据分析与 BI 域（Analytics / BI）报表、分析
- 规则引擎与策略引擎域（Rule Engine）规则表达、运营规则、博彩规则、动态执行
- 任务调度与异步处理域（Job / MQ） 定时任务、延迟任务
## 十三、后台与管理域（B 端系统）
- 后台管理域（Admin Console）用户管理、内容管理、配置管理
- 权限与操作审计域（Admin Auth / Audit）后台权限、操作留痕
- 发布与变更管理域（Release / Change）配置发布、回滚
- 后台管理域（Admin / Console）用户管理、内容管理、配置管理
- 组织与账号域（Org / Staff）运营人员、权限分组、多租户
- 发布与版本域（Release）配置发布、回滚、灰度控制
## 十四、产品形态总结
- 拉新 → 注册 → 合规校验 / 实名 / 年龄校验 → 内容 / 场景触达（推荐 / 直播 / 赛事 / 真人视讯） → 使用 → 付费 → 投注 / 游戏 → 锁单 → 结算 / 派奖 → 风控 → 留存 / 复购 → 申诉 / 仲裁 → 退出 / 注销 等等
- 充值 → 入账 → 钱包入账 → 投注 / 游戏下注 → 锁单 → 赛果 / 回合结果 → 结算 → 返奖 / 派奖 → 账本入账 → 对账 → 风控 → 合规审计 / 合规报送 等等
- 核心域：投注、赔率、玩法规则、真人视讯（桌台 / 荷官 / 牌局）、结算、返奖、合规、风控（投注风控 / 交易风控 / 业务风控 / 反洗钱）等
- 支撑域：支付、资产 / 钱包、订单 / 交易、内容、社交关系、消息通知、实时互动（房间 / 会话 / 连麦 / 弹幕）、礼物道具、虚拟经济、推荐分发、运营活动、客服与申诉、增长激励、对账与清结算等
- 通用域：用户、身份认证与权限、组织与角色、配置与参数中心、功能开关与灰度、日志与审计、规则 / 策略引擎、任务调度 / 消息机制、数据采集与分析、后台管理、多租户与账号体系、合规审计与追溯能力等
