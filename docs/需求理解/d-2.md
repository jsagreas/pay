# 第一部分：多币种钱包模块（主责）

## 一、需求背景与目标

> 来源：1-需求背景.png、2-需求目标.png、3-适用范围.png

- 平台面向多地区运营（越南、印尼、泰国等），需要支持多种法币和虚拟币
- 目标：构建多币种钱包体系，支持充值、提现、兑换、记录等完整资金链路
- 适用范围：C端用户（App/H5/Web），后端需提供对应的API支撑

## 二、钱包总体设计

> 来源：4-1钱包总体设计.png、4-1钱包总体设计2.png

### 2.1 钱包结构

- 钱包按**币种维度隔离**，每个币种独立余额与账本
- 当前支持币种：**BSB**（平台币）、**VND**（越南盾）、**IDR**（印尼盾）、**THB**（泰铢）
- BSB锚定关系：10 BSB = 1 USDT（即1 BSB = 0.1 USDT）
- 每个币种下有多个子钱包：
  - **中心钱包**：充值到账的主钱包，消费优先扣这里
  - **奖励钱包**：活动赠送、兑换赠送等奖金存放处，消费其次扣这里
  - **资金钱包** = 中心钱包余额 + 奖励钱包余额（对外展示的汇总余额）
  - 主播钱包、代理钱包（预留）、场馆钱包（预留）
- 金额展示规则：
  - VND/IDR：`.`作为千分位符，整数（无小数）
  - THB/BSB：`,`作为千分位符，保留2位小数
- **不支持跨币种混合展示与操作**

### 2.2 币种切换规则

> 来源：4-2币种切换规则.png

- 用户在个人中心/钱包页面可切换当前操作币种
- 切换后，所有页面（充值、提现、兑换、记录）联动切换到对应币种数据
- 每个币种独立的余额、充值方式、提现方式、记录

### 2.3 页面交互

> 来源：4-3页面交互.png

- 个人中心展示当前币种的资金钱包余额
- 点击余额区域可展开查看中心钱包和奖励钱包的分项余额
- 币种切换入口在钱包页面顶部

### 2.4 产品原型对照

> 来源：2-1个人中心.png、2-2个人中心.png、2-3个人中心.png、3-1~3-4充值币种切换.png

- 个人中心页面展示：当前币种、资金余额、充值/提现/兑换/记录四个入口
- 币种切换：下拉列表展示所有可用币种，带币种图标和名称
- 切换后整个钱包页面数据联动刷新

## 三、充值模块

### 3.1 充值方式总览

> 来源：5-1充值方式总览.png、5-1充值方式总览2.png、5-1充值方式总览3.png

- 充值方式分三大类：

| 类型 | 方式 | 承载页面 | 特点 |
|------|------|---------|------|
| 虚拟币 | USDT（TRC20/ERC20/BEP20） | 平台承载 | 平台生成充值地址，用户链上转账 |
| 银行 | 银行转账 | 三方承载 | 跳转三方支付页面 |
| 电子钱包/扫码 | QRIS/PromptPay等 | 三方承载 | 跳转三方或展示二维码 |

- 游客状态：仅可使用USDT充值
- 登录用户：解锁全部充值方式
- 充值方式的配置（名称、图标、档位、金额范围、关联通道）来自后台的支付配置模块（财务负责）

### 3.2 USDT充值

> 来源：5-2usdt充值.png、5-2usdt充值2.png、5-2usdt充值3.png

- 用户选择链类型（TRC20/ERC20/BEP20）
- 平台生成对应链的充值地址（可复制）
- 页面展示充值地址、最低充值金额、到账确认数
- 仅在BSB充值界面显示USDT→BSB的汇率换算
- 法币充值界面不显示汇率换算（直接按该法币金额充值）

### 3.3 银行充值

> 来源：5-3银行充值.png、5-3银行充值2.png

- 用户选择银行充值方式 → 选择/输入金额 → 提交后跳转三方支付页面
- 金额输入方式：快捷档位选择 或 自定义金额输入（在配置的范围内）
- 提交订单后由后端轮询匹配通道 → 创建通道订单 → 返回三方支付URL

### 3.4 扫码充值

> 来源：5-4扫码充值.png、5-4扫码充值2.png

- 类似银行充值，但最终展示形式为二维码
- 用户使用对应App扫码支付（如QRIS、PromptPay）

### 3.5 重复转账拦截规则

> 来源：5-5重复转账拦截规则.png、5-5重复转账拦截规则2.png

- 判断条件：同一UserID + 同一币种 + 同一金额，在短时间内
- 第1-2笔：弹窗警告提示"您近期有相同金额的充值"，用户可选择继续或取消
- 第3笔及以上：直接拦截，不允许继续提交
- 目的：防止用户误操作重复充值

### 3.6 额外奖金选择规则

> 来源：5-6额外奖金选择规则.png、5-6额外奖金选择规则2.png

- 充值时可选择参与活动获得额外奖金
- 活动奖金与充值金额联动：不同金额档位对应不同奖金比例
- 快捷充值标签上带角标标识（如"首充+20%"）
- 有门槛校验：充值金额需达到活动最低要求才能选择该奖金
- 赠送的奖金进入**奖励钱包**，需完成X倍流水后方可提现

### 3.7 充值页面交互

> 来源：5-7页面交互.png

- 充值流程：选择充值方式 → 选择/输入金额 → 选择奖金（可选）→ 提交订单
- 提交后展示订单详情页（订单号、金额、状态）

### 3.8 通用回调

> 来源：7-1通用回调.png、7-2通用回调.png、7-3通用回调.png、7-4通用回调.png

- 三方支付完成后回调平台
- 回调处理：验证签名 → 验证金额 → 订单状态更新 → 用户上账
- 需要幂等处理，防止重复回调导致重复上账

### 3.9 充值产品原型对照

> 来源：1-三方充值页面.png、4-1虚拟币充值.png、5-1~5-4切换充值方式.png、6-1选择奖金.png、6-2选择奖金.png、8兑换BSB.png

- 三方充值页面：展示充值金额、支付方式图标、跳转三方按钮
- 虚拟币充值页面：链类型选择tabs、充值地址（可复制）、注意事项
- 切换充值方式：底部弹出sheet，展示所有可用充值方式（图标+名称）
- 选择奖金：充值金额档位带角标，选中后下方显示奖金明细

## 四、兑换模块

> 来源：6兑换模块1.png、6兑换模块2.png

- **仅法币钱包有兑换入口**（VND/IDR/THB → BSB）
- BSB钱包没有兑换入口（BSB不能反向兑换为法币）
- 汇率统一口径：1 BSB = X 法币（如1 BSB = 2,300 VND）
- 兑换流程：输入法币金额 → 按汇率计算BSB数量 → 确认兑换
- 兑换赠送奖励（如额外10%），赠送部分进入奖励钱包
- 赠送部分需完成X倍流水后才可提现

### 4.1 兑换产品原型对照

> 来源：8兑换BSB.png

- 页面展示：当前法币余额、汇率、输入兑换金额、预计获得BSB数量、赠送奖励明细
- 确认按钮提交兑换

## 五、提现模块

### 5.1 提现总体规则

> 来源：7-1提现模块1.png、7-1提现模块2.png、7-1提现模块3.png、7-1提现模块4.png、7-1提现模块5.png

**KYC与提现的关系：**
- 未认证用户：点击提现时弹窗提示
  - "继续提现"：仅能使用USDT提现
  - "去认证"：跳转KYC认证流程，认证后解锁全部提现方式
- 已认证用户：直接进入提现方式选择页面

**提现方式的充值来源规则：**

| 充值来源 | 可用提现方式 |
|----------|------------|
| 法币充值 | 对应法币提现 |
| USDT充值 | USDT提现 |
| 法币 + USDT混合充值 | 法币提现 |
| 法币 → 兑换BSB | 法币提现 |

**提现限制：**

> 来源：7-5提现限制.png

- 每日提现次数限制
- 每日提现金额上限
- 单笔最低/最高金额限制
- 大额提现触发人工审核
- 异常频率触发风控

### 5.2 银行卡提现

> 来源：7-2银行卡提现.png

- 用户填写/选择银行卡账户信息（银行名称、卡号、持有人姓名）
- 持有人姓名 = KYC认证姓名（不可编辑）
- 首次填写后平台记录，后续复用
- 到账时效因国家而异

### 5.3 电子钱包提现

> 来源：7-3电子钱包提现.png、7-3电子钱包提现2.png

- 不同国家支持不同的电子钱包：
  - 越南：银行卡（T+1到账）
  - 印尼：GoPay/OVO/DANA/ShopeePay（即时到账）+ 银行卡
  - 泰国：PromptPay（当天到账）+ 银行卡
- 用户选择电子钱包类型 → 填写账户信息 → 输入提现金额 → 提交

### 5.4 BSB提现

> 来源：7-4bsb提现.png

- BSB钱包提现方式：USDT + 各国银行/电子钱包
- BSB提现时展示汇率换算（BSB → 对应法币/USDT的换算）
- 出款按出款汇率计算

### 5.5 提现页面交互

> 来源：7-6页面交互.png

- 提现流程：选择提现方式 → 填写/选择账户信息 → 输入金额 → 确认提交
- 提交后展示订单状态（待审核/处理中/已完成/已驳回）

### 5.6 提现触发认证原型

> 来源：9-1~9-4提现触发认证.png

- 未认证用户点提现 → 弹窗（两个按钮："继续提现"/"去认证"）
- "继续提现" → 进入USDT提现页面
- "去认证" → 跳转KYC页面

### 5.7 提现产品原型对照

## 七、非功能性需求

> 来源：9非功能性需求.png

- **强一致性**：账务流水与余额必须强一致，不允许出现余额与流水不一致的情况
- **幂等处理**：充值回调必须幂等，重复回调不重复上账
- **KYC强制校验**：提现必须校验KYC状态
- **提现账户白名单**：提现账户需经过验证
- **姓名强一致校验**：提现账户持有人姓名必须与KYC认证姓名一致
- **大额/异常频率审核**：触发人工审核机制

# 第二部分：币种配置模块（主责）

## 一、币种配置说明

> 来源：1-币种配置说明.png、1-币种配置说明2.png

- 币种配置是运营后台（B端）的管理功能
- 目的：统一管理平台支持的所有币种参数，服务于钱包、充值、提现、统计等场景
- 配置内容包括：币种基本信息、汇率参数、展示规则、状态控制

## 二、资金钱包体系

> 来源：2-1资金钱包体系.png ~ 2-1资金钱包体系7.png（共7张）

### 2.1 钱包类型与层级

| 钱包类型 | 说明 | 状态 |
|----------|------|------|
| 中心钱包 | 充值到账的主钱包 | 已确定 |
| 奖励钱包 | 活动赠送、兑换赠送的奖金 | 已确定 |
| 主播钱包 | 主播收益 | 预留 |
| 代理钱包 | 代理佣金 | 预留 |
| 场馆钱包 | 场馆/游戏锁定资金 | 预留 |

- **资金钱包** = 中心钱包 + 奖励钱包（用户可见的余额汇总）

### 2.2 消费扣款优先级

- 优先扣**中心钱包**，中心钱包不足时扣**奖励钱包**
- 混合投注时（部分中心+部分奖励），返奖需按金额占比分别计入对应钱包

### 2.3 各钱包资金来源与用途

| 钱包 | 资金来源 | 用途 |
|------|---------|------|
| 中心钱包 | 充值到账、人工加款 | 投注、提现、兑换 |
| 奖励钱包 | 活动赠送、兑换赠送、返奖 | 投注（需完成流水后可提现） |
| 主播钱包 | 直播收益 | 提现 |
| 代理钱包 | 代理佣金 | 提现 |

## 三、币种类型划分

> 来源：3-1币种类型划分.png

| 类型 | 示例 | 说明 |
|------|------|------|
| 法定货币（Fiat） | VND、IDR、THB | 与国家/地区绑定 |
| 加密货币（Crypto） | USDT、BTC、ETH | 区块链数字货币 |
| 平台货币（Platform） | BSB | 平台自定义虚拟币 |

## 四、平台基准币种

> 来源：3-2平台基准币种.png

- 推荐设置**USDT**为基准币种
- 基准币种一旦设置**不可更改**（影响全局计算逻辑）
- BSB锚定基准币种：10 BSB = 1 USDT
- 基准币种用途：
  - 活动奖金的统一计算基准
  - 多币种数据的统一统计换算
  - 财务报表的统一结算币种

## 五、后台配置页面

> 来源：3-3后台配置页面说明.png ~ 3-3后台配置页面说明5.png（共5张）

### 5.1 币种列表字段

| 字段 | 说明 |
|------|------|
| 币种代码 | 如VND、IDR、USDT、BSB |
| 币种名称 | 如越南盾、印尼盾 |
| 币种类型 | 法定货币/加密货币/平台货币 |
| 国籍/地区 | 币种关联的国家（法币才有） |
| 币种图标 | SVG/PNG格式，≤5KB |
| 千分位符号 | `.`或`,` |
| 小数点符号 | `,`或`.` |
| 小数位数 | 法币0-2位，加密货币最多8位 |
| 实时汇率 | 三方API获取的实时汇率 |
| 平台汇率 | 平台计算后的汇率 |
| 汇率浮动 | 百分比，影响入款/出款汇率 |
| 入款汇率 | 实时汇率 × (1 + 浮动%) |
| 出款汇率 | 实时汇率 × (1 - 浮动%) |
| 偏值 | 偏差百分比阈值 |
| 状态 | 启用/禁用 |

### 5.2 编辑弹窗

- 可编辑字段：汇率浮动（%）、币种图标、偏值（%）、状态（启用/禁用）
- 币种代码、名称、类型等基础信息创建后不可修改
- 禁用币种后：C端不可见该币种，不可充值/提现/兑换

- 启用/禁用：记录状态变更

- 操作日志tab：展示该模块的所有操作记录（操作人、操作类型、操作时间、变更详情）

## 六、汇率规则

### 6.1 汇率应用场景

> 来源：4-1汇率应用场景.png

- 充值：法币充值按1:1（无汇率换算），USDT充值BSB需按入款汇率换算
- 提现：BSB提现需按出款汇率换算为法币/USDT
- 兑换：法币→BSB按平台汇率换算
- 统计：财务报表需按基准币种汇率统一换算

### 6.2 汇率规则说明

> 来源：4-2汇率规则说明1.png、4-2汇率规则说明2.png

- **实时汇率**：从3家三方汇率API获取，取均值
- **平台入款汇率** = 实时汇率 × (1 + 汇率浮动%)
  - 浮动为正：用户充值时平台多收（对平台有利）
- **平台出款汇率** = 实时汇率 × (1 - 汇率浮动%)
  - 浮动为正：用户提现时平台少付（对平台有利）
- **偏差值计算**：|实时汇率 - 当前平台汇率| / 实时汇率 × 100%
- **偏差值触发更新**：偏差值 ≥ 设定的偏值阈值时 → 自动更新该币种的平台汇率 → 生成汇率变动日志

### 6.3 汇率展示规则

> 来源：4-3汇率展示.png

- 法币汇率：保留2位小数
- 加密货币汇率：保留8位小数
- 展示格式：1 [基准币种] = X [目标币种]

### 6.4 汇率日志

> 来源：4-4汇率日志.png、4-4汇率日志2.png、4-4汇率日志3.png

- 记录每次汇率变动：
  - 触发原因（定时更新/手动刷新/偏差触发）
  - 变动前汇率 → 变动后汇率
  - 变动时间
  - 操作人（系统自动/管理员）
- 支持按币种、时间范围筛选查询

> 来源：2-3财务管理-充值记录-1充值流程.png


# 第四部分：跨模块依赖关系总结

## 一、财务 → 钱包（财务调用钱包的场景）

| 场景 | 触发时机 | 调用的钱包能力 | 数据流向 |
|------|---------|--------------|---------|
| 充值成功上账 | 充值回调验证通过 | CreditWallet（中心钱包） | 财务→钱包：用户ID+币种+金额+订单号 |
| 活动奖金发放 | 充值成功且选择了活动 | CreditWallet（奖励钱包） | 财务→钱包：用户ID+币种+奖金金额 |
| 充值补单通过 | 补单审核通过 | CreditWallet | 同充值上账 |
| 人工加款通过 | 加款审核通过 | CreditWallet × 4个钱包 | 财务→钱包：4个钱包各自金额 |
| 人工减款通过 | 减款审核通过 | DebitWallet × 4个钱包 | 财务→钱包：4个钱包各自减扣金额 |
| 提现创建 | 用户提交提现订单 | FreezeBalance | 钱包冻结对应金额 |
| 提现驳回/出款失败 | 审核驳回或三方出款失败 | UnfreezeBalance | 钱包解冻已冻结金额 |
| 出款成功 | 三方出款成功回调 | ConfirmDebit | 钱包正式扣减已冻结金额 |
| 查询用户余额 | 人工加减款页面 | GetBalance | 展示当前各钱包余额 |

## 二、钱包 → 财务（钱包调用财务的场景）

| 场景 | 触发时机 | 调用的财务能力 | 数据流向 |
|------|---------|--------------|---------|
| 获取充值方式 | C端进入充值页面 | 支付配置-充值方式列表 | 返回：方式名称/图标/档位/金额范围/关联通道 |
| 获取提现方式 | C端进入提现页面 | 支付配置-提现方式列表 | 返回：方式名称/图标/费率/关联通道 |
| 充值匹配通道 | 用户提交充值订单 | 通道轮询匹配 | 钱包→财务：币种+充值方式+金额 → 返回：通道信息+三方支付URL |
| 提现提交出款 | 提现审核全部通过 | 通道出款 | 钱包→财务：订单+金额+账户信息 → 返回：出款结果 |

## 三、财务 → 币种配置（财务调用币种配置的场景）
