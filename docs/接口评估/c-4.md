## 架构前提（基于工程分析确认）

1. **单服务多域**：ser-wallet 包含「币种配置」+「钱包」两个域，同一服务进程；两域之间是内部函数调用（非RPC）
2. **IDL定义**：统一在 `common/idl/ser-wallet/service.thrift` + 域 thrift 文件，生成到 `common/pkg/kitex_gen/ser_wallet/`
3. **网关路由**：B端接口注册到 gate-back `/admin/api/wallet/...`；C端接口注册到 gate-font `/api/wallet/...`
4. **Handler模式**：~95% 使用 `rpc.Handler(ctx, c, rpc.WalletClient().Method)` 一行委托模式
5. **审计日志**：所有写操作调用 `ser-blog.AddActLog`（oneway），遵循项目已有标准模式
6. **错误码**：按项目规范分配新段（如 `6022xxx`，具体段号待确认）
7. **上下文传播**：8个 TTHeader 字段自动传播（TraceID/ClientIP/UserToken/UserID/UserName/Language/DeviceID/DeviceType）
8. **RPC客户端**：在 `common/rpc/rpc_client.go` 新增 `WalletClient()` sync.Once 懒加载单例，ETCD 服务名 `wallet_service`

### 2.2 RPC供其他服务调用（主要供财务管理模块调用）

> **核心模式：资金冻结/解冻/确认扣款**
> 提现链路中，钱包与财务之间的资金状态流转是关键：
> - 创建提现订单 → **冻结**
> - 审核拒绝 / 出款失败 → **解冻返还**
> - 出款成功 → **确认扣除冻结金额**
> 此模式确保多步异步提现流程中资金不被重复消费也不丢失

| 编号 | 接口名 | 优先级 | 调用方 | 说明 | 依据 |
|------|--------|--------|--------|------|------|
| W1 | GetUserWalletBalance | P0 | 财务管理 | 查询指定用户各子钱包当前余额（人工加减款弹窗展示用） | 财务需求E5：人工加减款需展示各钱包余额 |
| W2 | CreditDeposit | P0 | 财务管理 | 充值成功上账：充值金额→中心钱包，奖金→奖励钱包（分别入账） | 财务需求E3：充值流程→上分 |
| W3 | FreezeForWithdrawal | P0 | 财务管理 | 提现冻结：冻结用户钱包对应金额 | 财务需求E4：提现流程→冻结 |
| W4 | UnfreezeForWithdrawal | P0 | 财务管理 | 提现解冻：风控/财务审核拒绝或出款失败时返还冻结金额 | 财务需求E4：拒绝→解冻 |
| W5 | ConfirmWithdrawalDeduct | P0 | 财务管理 | 提现确认扣款：出款成功后最终扣除已冻结金额 | 财务需求E4：出款成功→扣款 |
| W6 | ManualCredit | P0 | 财务管理 | 人工加款：按钱包类型（中心/奖励/代理/主播）分别加款 | 财务需求E5：人工加款 |
| W7 | ManualDebit | P0 | 财务管理 | 人工减款：按钱包类型分别扣款（余额不足扣实际余额） | 财务需求E5：人工减款 |

```

人工修正：财务创建审核→通过→钱包ManualCredit/ManualDebit/CreditCorrection
```

## 七、首版修正说明

对比首版，以下接口认识有调整：

| 首版接口 | 二版调整 | 原因 |
|---------|---------|------|
| 充值回调(C端) | 移除 → 归财务侧接收回调后RPC通知钱包 | 三方回调不是C端用户发起的请求 |
| 提现回调(C端) | 移除 → 同上 | 出款回调同理 |
| KYC状态查询(C端独立接口) | 合并入 GetWithdrawalInfo | 一次请求返回KYC状态+可用方式，减少客户端调用次数 |
| 钱包余额变更(笼统1个) | 拆分为8个具体RPC接口 | 上账/冻结/解冻/扣款/加款/减款/补单是不同业务语义，不能混为一谈 |
| 汇率查询(混为一个) | 拆分为B端汇率日志 + RPC汇率查询 + 内部定时同步 | 三者场景不同：B端查日志、外部查汇率、内部同步更新 |
| 财务管理(~25-28) | 修正为~42 | 按项目已有CRUD拆分模式，每个实体独立Create/Update/Query/Toggle |
