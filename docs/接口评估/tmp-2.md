## 五、模块间依赖关系

```
┌─────────────┐       ┌─────────────────┐       ┌─────────────┐
│  币种配置    │◄──────│   多币种钱包     │◄──────│  财务管理    │
│  (基础设施)  │ 汇率  │   (核心业务)     │ 入账  │  (运营管理)  │
│             │ 币种  │                 │ 冻结  │             │
│             │ 精度  │                 │ 解冻  │             │
│             │       │                 │ 扣款  │             │
│             │       │                 │ 加减款│             │
└─────────────┘       └─────────────────┘       └─────────────┘
     基础层                 业务层                  管理层
```

- **币种配置 → 钱包**：钱包的所有金额操作依赖币种精度、汇率计算
- **财务 → 钱包**：充值入账、提现冻结/解冻/扣款、人工加减款均通过内部RPC调用钱包
- **钱包 → 财务**：财务弹窗需读取钱包余额
- **钱包 → 币种配置**：兑换/充值/提现均需获取汇率和币种信息

## 六、说明与保留事项

1. **预评审状态**：以上接口清单基于当前产品原型和需求文档，尚未通过正式需求评审，细节可能在跨部门评审后发生变更（新增/删除/调整）
2. **粒度说明**：这是粗粒度估算，评审后数量可能浮动，但量级不会有大的偏差，核心骨架就是这些功能点
3. **定时任务**：汇率偏差检测更新、对账等可能以定时任务形式实现而非暴露为接口，未计入接口数
4. **内部RPC vs HTTP**：C端接口走HTTP（Hertz），模块间联动走RPC（Kitex），上述估算包含两种类型
5. **核心职责边界**：币种配置 + 多币种钱包 = 我方负责的 ~29 个接口；财务管理模块为他方负责，需清楚依赖关系但不负责实现
6. **与第一版差异**：本版相比 tmp-1.md 做了更细致的拆分，钱包模块从 12~15 细化到 ~21（补充了充值回调、重复转账检测、优惠活动、提现限额等C端接口），财务模块从 18~22 上调到 ~28（补充了更多人工修正和统计类接口）

## 评估方法论

### 四维分类

| 维度 | 说明 | 网关/协议 |
|------|------|----------|
| **C端 HTTP** | 面向终端用户（iOS/Android/H5） | gate-font，Hertz HTTP |
| **B端 HTTP** | 面向后台运营/管理人员 | gate-back，Hertz HTTP |
| **RPC 供他方** | ser-wallet 暴露给其他微服务调用的 RPC 接口 | Kitex Thrift |
| **RPC 调他方** | ser-wallet 主动调用其他微服务的 RPC 接口 | Kitex Thrift |

### 三级优先级

| 优先级 | 定义 | 标记 |
|--------|------|------|
| **核心（Must）** | 需求文档明确要求、业务链路绕不过的接口 | ★★★ |
| **应该有（Should）** | 需求暗示或业务逻辑合理需要、不影响核心链路但缺失会降低体验 | ★★☆ |
| **可选（Could）** | 视评审结果决定、可能被合并或由定时任务替代 | ★☆☆ |

                            ┌──────────────┐
                            │   ser-kyc    │
                            │  (KYC状态)   │
                            └──────┬───────┘
                                   │ O-1: QueryKycStatus
                                   ▼
┌──────────────┐    R-8~10     ┌──────────────────────┐    R-1~6      ┌──────────────┐
│   其他模块    │◄─────────────│      ser-wallet       │◄─────────────│  财务管理模块  │
│  (游戏/直播   │  币种/汇率    │  ┌────────────────┐  │  入账/冻结     │  (通道/审核    │
│   礼物/投注)  │              │  │  钱包核心       │  │  解冻/扣冻     │   支付配置     │
│              │  R-1,2,7     │  │  (余额/账变)   │  │  查余额        │   人工修正)    │
│              │─────────────►│  ├────────────────┤  │              │              │
│              │  扣款/入账    │  │  币种配置       │  │  O-3~5        │              │
│              │              │  │  (汇率/币种)   │  │─────────────►│              │
└──────────────┘              │  └────────────────┘  │  支付方式      └──────┬───────┘
                              └──────────┬───────────┘  通道匹配             │
                                   │ O-2: GetUserInfo    代付请求             │
                                   ▼                                    回调 │
                            ┌──────────────┐                    ┌───────────▼──┐
                            │   ser-user   │                    │   三方支付通道  │
                            │  (用户信息)   │                    │  (代收/代付)   │
                            └──────────────┘                    └──────────────┘

**调用方向说明：**
- `→` 箭头指向被调用方
- 财务模块 → 钱包：充值成功入账、提现冻结/解冻/扣冻、人工加减款、查余额（6个RPC）
- 钱包 → 财务模块：获取支付配置、通道匹配、发起代付（3个RPC）
- 钱包 → ser-kyc：提现前KYC校验（1个RPC）
- 钱包 → ser-user：获取用户信息（1个RPC）
- 其他模块 → 钱包：消费扣款、奖励入账、查币种/汇率（涉及R-1~R-10）
