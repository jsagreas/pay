### 2.5 B端 HTTP 接口说明

> **重要分析结论**：钱包模块本身可能不需要独立的B端HTTP接口。
>
> 原因：所有B端钱包操作均由财务模块发起，通过RPC调用钱包：
> - 查看余额 → 财务模块B端调用钱包 RPC GetBalance
> - 加款/减款 → 财务模块B端调用钱包 RPC CreditWallet / DebitWallet
> - 冻结/解冻 → 财务模块B端调用钱包 RPC FreezeBalance / UnfreezeBalance
> - 充值/提现记录 → 财务模块B端自行管理
>
> 若评审中确认需要钱包独立的B端管理页面（如运营查看用户钱包流水），则需增加：
