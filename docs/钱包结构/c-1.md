## 1. 设计方法与依据

### 1.1 目录结构从哪里来

```
工程现有服务（ser-app / ser-buser / ser-item）的实际目录结构
    ↓ 提炼统一模式
8层标准骨架（cfg → enum → errs → cache → gen → gorm_gen → rep → ser）
    ↓ 叠加业务规模
ser-wallet 的 10 张表、42 个接口、9 组枚举
    ↓ 映射
每层需要哪些文件、文件怎么命名、文件放什么职责
    ↓
最终目录树
```

### 1.2 结构设计的三条原则

| 原则 | 说明 |
|------|------|
| **遵循现有工程规范** | 目录划分、文件命名、代码组织方式与 ser-app/ser-buser/ser-item 保持一致 |
| **一层一职责** | cfg=初始化、enum=枚举、errs=错误码、cache=缓存、rep=数据访问、ser=业务逻辑 |
| **按实体/域拆分文件** | rep 按数据库表实体拆分；ser 按业务域拆分；enum 按业务域分组 |

### 1.3 输入汇总

| 来源 | 影响目录结构的关键数据 |
|------|---------------------|
| 表结构设计/c-1.md | 10 张表 → 决定 gorm_gen、rep 层各需要多少文件 |
| IDL设计/c-1.md | 42 个接口 / 7 个 IDL 文件 → 决定 ser 层按哪些域拆分 |
| 接口评估/c-1.md | C端22 + B端5 + RPC15 → handler.go 的方法规模 |
| 工程认知/c-1.md | 标准 8 层目录结构、命名规范、错误码前缀 6022 |
| 技术栈选择/c-1.md | 依赖组件 → go.mod 需要引入的包 |

## 3. 根目录文件说明

### 3.1 各文件职责

| 文件 | 职责 | 参照 | 关键内容 |
|------|------|------|---------|
| **main.go** | 服务启动入口 | ser-item/main.go | 初始化日志 → 初始化ETCD → 初始化MySQL/Redis → 创建Service实例 → 启动Kitex RPC Server |
| **handler.go** | IDL方法委派 | ser-app/handler.go | 定义 `WalletServiceImpl` 结构体，持有各 `*ser.XxxService`，实现 IDL 定义的 42 个方法，每个方法一行委派 |
| **go.mod** | 模块依赖 | ser-app/go.mod | `module ser-wallet`，依赖 common 模块 + `shopspring/decimal`（唯一新增依赖） |
| **go.sum** | 依赖校验 | — | 自动生成，不手动修改 |
| **.gitignore** | Git忽略 | ser-app/.gitignore | 忽略 IDE 配置、编译产物、日志文件等 |
| **.golangci.yml** | 代码检查 | ser-app/.golangci.yml | 统一的 lint 规则配置 |

### 3.2 main.go 初始化顺序

```
main()
  ├── 1. lg.InitKLog()                    # 日志
  ├── 2. etcd.InitEtcd()                  # ETCD连接
  ├── 3. cfg.InitMySQL()                  # MySQL连接 + query.SetDefault
  ├── 4. cfg.InitRedis()                  # Redis连接
  ├── 5. 创建各 Service 实例               # NewWalletService / NewDepositService / ...
  ├── 6. walletservice.NewServer(impl)    # 构建Kitex Server
  └── 7. srv.Run()                        # 启动监听
```

### 3.3 handler.go 结构体设计

```
WalletServiceImpl
  ├── walletService      *ser.WalletService          # 钱包核心（3 个方法）
  ├── depositService     *ser.DepositService          # 充值（6 个方法）
  ├── withdrawalService  *ser.WithdrawalService       # 提现（7 个方法）
  ├── exchangeService    *ser.ExchangeService         # 兑换（4 个方法）
  ├── currencyService    *ser.CurrencyService         # 币种管理（5 个方法）
  ├── rewardService      *ser.RewardService           # 奖励（2 个方法）
  ├── enumService        *ser.EnumService             # 枚举（1 个方法）
  └── （RPC外部调用方法分散在上述 Service 中）        # 15 个 RPC 方法
```

> 参照 ser-app：handler.go 只做委派，不含任何业务逻辑。每个 IDL 方法对应一行 `return s.xxxService.Method(ctx, req)`。

## 4. internal 各层详解

### 4.1 cfg/ — 初始化层

| 文件 | 职责 | 对标 |
|------|------|------|
| mysql.go | 从 ETCD 读取 TiDB 连接参数 → 初始化 GORM DB → `query.SetDefault(db)` | ser-app/internal/cfg/mysql.go |
| redis.go | 从 ETCD 读取 Redis 连接参数 → 初始化 Redis 客户端 | ser-app/internal/cfg/redis.go |

**要点**：
- 使用 `sync.Once` 保证单例
- 数据库名称从 `namesp.EtcdWalletDb`（需在 common/pkg/consts/namesp/ 注册）读取
- 连接参数统一走 ETCD，不硬编码

---

### 4.2 enum/ — 枚举层

| 文件 | 包含的枚举定义 | 枚举数量 |
|------|--------------|---------|
| enum_registry.go | 枚举注册表 `EnumsOptionsRegistry`，统一对外查询入口 | — |
| wallet.go | `WalletType`（中心/奖励/冻结/法币/虚拟币）、`CurrencyCode`（VND/IDR/THB/USDT/BSB） | 2 组 |
| deposit.go | `DepositStatus`（待支付/成功/失败/已关闭）、`DepositMethod`（银行转账/电子钱包/USDT） | 2 组 |
| withdrawal.go | `WithdrawalStatus`（待审核/审核通过/处理中/成功/失败/拒绝）、`WithdrawalMethod`（银行卡/电子钱包/USDT）、`WithdrawalAccountType`（银行卡/电子钱包/USDT地址） | 3 组 |
| exchange.go | `ExchangeStatus`（成功/失败） | 1 组 |
| transaction.go | `LogType`（充值/提现/兑换/奖励/人工加款/人工减款/冻结/解冻...） | 1 组 |

**对标参照**：
- ser-app/internal/enum/banner.go — 每个文件定义一组业务常量
- ser-app/internal/enum/enum_registry.go — 统一注册表模式

**枚举注册表结构**：

```
EnumsOptionsRegistry = map[string]func() []EnumOption{
    "wallet_walletType":            GetWalletTypeOptions,
    "wallet_currencyCode":          GetCurrencyCodeOptions,
    "wallet_depositStatus":         GetDepositStatusOptions,
    "wallet_depositMethod":         GetDepositMethodOptions,
    "wallet_withdrawalStatus":      GetWithdrawalStatusOptions,
    "wallet_withdrawalMethod":      GetWithdrawalMethodOptions,
    "wallet_withdrawalAccountType": GetWithdrawalAccountTypeOptions,
    "wallet_exchangeStatus":        GetExchangeStatusOptions,
    "wallet_logType":               GetLogTypeOptions,
}
```

### 4.3 errs/ — 错误码层

| 文件 | 职责 |
|------|------|
| code.go | 按业务模块划分错误码段，前缀 `6022` |

**错误码分段规划**：

| 范围 | 模块 | 示例 |
|------|------|------|
| 6022000 ~ 6022099 | 钱包通用 | 参数错误、余额不足、币种不存在 |
| 6022100 ~ 6022199 | 充值 | 订单不存在、通道异常、重复充值 |
| 6022200 ~ 6022299 | 提现 | 限额超出、KYC未通过、账户信息缺失 |
| 6022300 ~ 6022399 | 兑换 | 汇率已过期、兑换金额不足 |
| 6022400 ~ 6022499 | 币种配置 | 基准币种已设置、币种已禁用 |
| 6022500 ~ 6022599 | 预留 | — |

> 对标 ser-app/internal/errs/code.go：`6011000 + iota` 起始，每个模块间隔 100。

### 4.4 cache/ — 缓存层

| 文件 | 缓存内容 | 缓存策略 |
|------|---------|---------|
| wallet.go | 用户各币种余额 | 读时缓存、余额变更时主动失效 |
| currency.go | 币种配置列表、汇率信息 | 定时刷新 + B端编辑时主动失效 |

**对标参照**：
- ser-app/internal/cache/banner.go — `GetList` / `SetList` / `ClearAll` 三方法模式
- 缓存 key 通过 `redis_key.GetKey()` 统一构造，前缀注册在 common/pkg/consts/redis_key/

### 4.5 gen/ — 代码生成入口

| 文件 | 职责 |
|------|------|
| gorm_gen.go | `go run` 执行，连接 ETCD → 获取数据库名 → 调用 `mysql_gen.GenCodeWithAll()` → 自动扫描所有表 → 生成 model/query/repo |

**注意**：当前 ser-wallet 的 gorm_gen.go 内容引用了 `namesp.EtcdAppDb`（从 ser-app 复制而来），编码阶段需改为 `namesp.EtcdWalletDb`。

### 4.6 rep/ — 数据访问层

| 文件 | 对应表 | 核心职责 |
|------|-------|---------|
| wallet.go | wallet | 余额查询、CAS余额更新（`WHERE available >= amount`）、按用户+币种查询 |
| transaction_log.go | transaction_log | 流水写入、按用户+类型分页查询、幂等检查（`order_id + log_type`） |
| deposit_order.go | deposit_order | 订单创建、状态更新、按用户分页查询、重复充值检测、超时订单批量查询 |
| withdrawal_order.go | withdrawal_order | 订单创建、状态流转更新、按用户分页查询、今日已提现金额汇总 |
| withdrawal_account.go | withdrawal_account | 账户保存/更新、按用户查询 |
| currency_config.go | currency_config | 全量查询、单个查询、汇率更新、启用/禁用 |
| exchange_record.go | exchange_record | 记录写入、按用户分页查询 |
| reward_record.go | reward_record | 记录写入、按用户分页查询 |
| rate_change_log.go | rate_change_log | 日志写入、按币种分页查询 |
| system_config.go | system_config | 按key查询、写入（基准币种设置等） |

**设计要点**：
- 每个文件定义一个 `XxxRepo struct{}` + `NewXxxRepo() *XxxRepo`
- 方法接收 `context.Context`，返回 `(result, error)`
- 查询通过 `query.Xxx`（gorm/gen 生成的类型安全查询器）构建
- 复杂筛选条件定义 `XxxFilter struct`（参照 ser-app/internal/rep/banner.go 的 `BannerFilter`）
- 分页查询统一使用 `dto.PageResult` 封装

### 4.7 ser/ — 业务逻辑层

| 文件 | 业务域 | 承载的接口方法 | 依赖的 rep |
|------|-------|-------------|-----------|
| wallet_service.go | 钱包核心 | GetUserBalance、GetWalletOverview、GetCurrencyList | wallet, currency_config |
| deposit_service.go | 充值 | GetDepositMethods、CreateDepositOrder、GetUsdtPayInfo、DepositCallback、CheckRepeatDeposit、PageDepositRecord | deposit_order, wallet, transaction_log, currency_config |
| withdrawal_service.go | 提现 | GetWithdrawalMethods、CreateWithdrawalOrder、SaveWithdrawalAccount、GetWithdrawalAccount、GetWithdrawalLimit、PageWithdrawalRecord、DetailWithdrawalOrder | withdrawal_order, withdrawal_account, wallet, transaction_log, currency_config |
| exchange_service.go | 兑换 | GetExchangeRate、ExecuteExchange、GetExchangePreview、PageExchangeRecord | exchange_record, wallet, transaction_log, currency_config |
| currency_service.go | B端币种管理 | PageCurrencyConfig、UpdateCurrencyConfig、SetBaseCurrency、GetRealtimeRate、PageRateChangeLog | currency_config, rate_change_log, system_config |
| reward_service.go | 奖励 | PageRewardRecord、（RPC: RewardCredit） | reward_record, wallet, transaction_log |
| enum_service.go | 枚举 | GetEnumOptions | — (调用 enum 包) |

**设计要点**：
- 每个文件定义一个 `XxxService struct` + `NewXxxService() *XxxService`
- 结构体持有依赖的 `*rep.XxxRepo` 和 `*cache.XxxCache`
- 跨域操作（如充值入账需要 wallet + transaction_log + deposit_order 三表联动）在 Service 层编排事务
- RPC 外部调用方法分散在对应的业务 Service 中（如 `RpcDebitBalance` 在 wallet_service.go）

**Service → Repo 依赖关系图**：

```
wallet_service ──→ wallet rep
    │              currency_config rep
    │              wallet cache
    │
deposit_service ──→ deposit_order rep
    │               wallet rep
    │               transaction_log rep
    │               currency_config rep
    │
withdrawal_service ──→ withdrawal_order rep
    │                   withdrawal_account rep
    │                   wallet rep
    │                   transaction_log rep
    │                   currency_config rep
    │
exchange_service ──→ exchange_record rep
    │                wallet rep
    │                transaction_log rep
    │                currency_config rep
    │
currency_service ──→ currency_config rep
    │                rate_change_log rep
    │                system_config rep
    │                currency cache
    │
reward_service ──→ reward_record rep
    │              wallet rep
    │              transaction_log rep
    │
enum_service ──→ (enum 包，不依赖 rep)
```

## 5. gorm_gen 自动生成层详解

### 5.1 生成机制

```
开发者运行 go run internal/gen/gorm_gen.go
    ↓
连接 ETCD → 获取数据库连接信息
    ↓
调用 mysql_gen.GenCodeWithAll(dbName, "ser-wallet", "")
    ↓
扫描数据库全部表
    ↓
自动生成 3 类文件到 internal/gorm_gen/
```

### 5.2 三个子目录的职责

| 子目录 | 作用 | 文件命名 | 文件数量 |
|--------|------|---------|---------|
| model/ | Go 结构体，字段与数据库列一一映射 | `{表名}.gen.go` | 10 个（每表一个） |
| query/ | 类型安全的查询构建器 + `SetDefault` 入口 | `{表名}.gen.go` + `gen.go` | 11 个（10 + 1） |
| repo/ | 表级仓储封装（基础 CRUD） | `{表名去下划线}r/{表名去下划线}r.go` | 10 个目录 / 10 个文件 |

### 5.3 repo 目录命名规则

repo 的目录名和文件名遵循 **表名去下划线 + 后缀 `r`** 的命名规则：

| 表名 | repo 目录 | repo 文件 |
|------|----------|----------|
| wallet | walletr/ | walletr.go |
| transaction_log | transactionlogr/ | transactionlogr.go |
| deposit_order | depositorderr/ | depositorderr.go |
| withdrawal_order | withdrawalorderr/ | withdrawalorderr.go |
| withdrawal_account | withdrawalaccountr/ | withdrawalaccountr.go |
| currency_config | currencyconfigr/ | currencyconfigr.go |
| exchange_record | exchangerecordr/ | exchangerecordr.go |
| reward_record | rewardrecordr/ | rewardrecordr.go |
| rate_change_log | ratechangelogr/ | ratechangelogr.go |
| system_config | systemconfigr/ | systemconfigr.go |

> 命名依据：ser-app 的 `bannerr/bannerr.go`、`homecategoryr/homecategoryr.go`；ser-buser 的 `sysuserr/sysuserr.go`。

### 5.4 关键约束

- **禁止手动修改** gorm_gen/ 下的任何文件 — 重新运行生成脚本会覆盖
- 自定义查询逻辑放在 **rep/** 层，通过 `query.Xxx` 构建

## 9. 与现有服务的结构对比

### 9.1 对照表

| 对比维度 | ser-app | ser-buser | ser-item | **ser-wallet** |
|---------|---------|-----------|----------|---------------|
| 数据库表 | 2 | 5 | 2 | **10** |
| IDL方法 | 20 | 28 | ~15 | **42** |
| rep 文件 | 2 | 3 | 2 | **10** |
| ser 文件 | 5 | 4 | 3 | **7** |
| enum 文件 | 4 | 3 | 1 | **6** |
| cache 文件 | 1 | 0 | 0 | **2** |
| test 文件 | 0 | 3 | 4 | **6** |
| domain 文件 | 0 | 0 | 8 | **0** (预留) |
| 总文件数(手动) | ~14 | ~20 | ~18 | **~42** |

### 9.2 规模定位

ser-wallet 在当前 12 个微服务中属于 **中大型服务**：
- 表数量（10 张）是 ser-app（2 张）的 5 倍
- 接口数量（42 个）与 ser-live（39 个）相当
- 涉及资金操作，需要更严格的流水、缓存、错误码管理

### 9.3 可选扩展：domain/ 层

ser-item 使用了 `internal/domain/` 存放业务辅助逻辑（validator、mapper、helper 等）。ser-wallet 当前设计中未包含 domain/，原因：

- 目前 Service 层的业务逻辑可以在各 `*_service.go` 内完成
- 如果后续 Service 文件膨胀（单文件超过 500 行）或出现跨 Service 的公共逻辑，可按需引入：

```
internal/domain/                    # 按需引入
    ├── decimal_helper.go           #   金额计算辅助（shopspring/decimal 封装）
    ├── balance_validator.go        #   余额校验规则
    └── rate_calculator.go          #   汇率换算逻辑
```
