## 三、逐层详解

### 3.1 根目录文件

| 文件 | 职责 | 工程规范参照 |
|------|------|------------|
| `main.go` | 服务启动入口：初始化日志 → 初始化 ETCD → 读取端口 → 创建 Service 实例 → 启动 Kitex Server | ser-item/main.go 完全一致 |
| `handler.go` | RPC 方法分发层：定义 `WalletServiceImpl` 结构体，每个 Thrift 方法一行委托到 `internal/ser/` | ser-item/handler.go 完全一致 |
| `go.mod` | 模块声明 `module ser-wallet`，依赖 `common/pkg`、`common/rpc`、`shopspring/decimal` | 同其他 ser-xxx |
| `sql/01.table.sql` | 14 张表的完整 CREATE TABLE DDL | ser-item/sql/1.table.sql、ser-buser/sql/01.table.sql |

#### main.go 骨架逻辑

```
lg.InitKLog()
etcd.InitEtcd()
port := flag.String("port", etcd.DirectGet(ctx, namesp.EtcdWalletPort), "端口号")
flag.Parse()

// 创建所有 Service 实例
currencyService  := ser.NewCurrencyService()
accountService   := ser.NewAccountService()
depositService   := ser.NewDepositService()
withdrawService  := ser.NewWithdrawService()
exchangeService  := ser.NewExchangeService()
recordService    := ser.NewRecordService()
betService       := ser.NewBetService()
walletRpcService := ser.NewWalletRpcService()
rateSyncService  := ser.NewRateSyncService()
enumService      := ser.NewEnumService()

// 启动汇率定时同步
rateSyncService.StartCronSync()

// 启动 Kitex Server
srv := walletservice.NewServer(
    &WalletServiceImpl{...},
    rpc.InitRpcServerParams(namesp.EtcdWalletService, utils.S2Int(ctx, *port))...,
)
srv.Run()
```

#### handler.go 骨架逻辑

```go
type WalletServiceImpl struct {
    currencyService  *ser.CurrencyService
    accountService   *ser.AccountService
    depositService   *ser.DepositService
    withdrawService  *ser.WithdrawService
    exchangeService  *ser.ExchangeService
    recordService    *ser.RecordService
    betService       *ser.BetService
    walletRpcService *ser.WalletRpcService
    enumService      *ser.EnumService
}

// 每个方法一行委托，零逻辑。例如：
func (s *WalletServiceImpl) CurrencyList(ctx, req) (resp, err) {
    return s.currencyService.CurrencyList(ctx, req)
}
func (s *WalletServiceImpl) WalletCredit(ctx, req) (resp, err) {
    return s.walletRpcService.WalletCredit(ctx, req)
}
```

| 6022800 ~ 6022899 | 兑换 | ExchangeRateExpired, ExchangeAmountTooSmall |

### 3.6 internal/gorm_gen/ — 自动生成代码

> 此目录全部由 gen/gorm_gen.go 自动生成，**禁止手动修改**。

| 子目录 | 内容 | 文件数量 |
|--------|------|---------|
| `model/` | 每张表对应一个 `.gen.go`，定义 Go 结构体 | 14 个（与表数一致） |
| `query/` | 每张表对应一个 `.gen.go`，提供类型安全查询方法 + 一个 `gen.go` 全局入口 | 15 个 |
| `repo/` | 每张表一个子目录，提供 ~20 个标准 CRUD 方法 | 11 个子目录（Must + Should 表） |

**repo 提供的标准方法**（每张表都有）：

```
Create / BatchCreate / Update / BatchUpdate / BatchSave
UpdateByIds / UpdateByConds / UpdateFields
DeleteById / GetOneById / GetListByIds / GetListByConds
GetPageByConds / Count / Exists
```

### 3.7 internal/cache/ — Redis 缓存封装

| 文件 | 职责 | 缓存策略 |
|------|------|---------|
| `currency_cache.go` | 币种列表缓存、汇率缓存 | Cache-Aside，TTL 3~5 分钟，变更时删缓存 |
| `dedup.go` | 用户操作防重（充值/提现/兑换） | Redis SetNX，TTL 10 秒 |

**注意**：余额不缓存（ADR-02），此目录仅处理配置类缓存和防重。

### 3.8 internal/domain/ — 领域辅助对象

| 文件 | 职责 | 被谁调用 |
|------|------|---------|
| `amount.go` | shopspring/decimal 的工具封装：NewFromString、Add/Sub/Mul/Div、格式化输出、精度截断 | 所有涉及金额计算的 ser/ 文件 |
| `rate_calculator.go` | 入款汇率/出款汇率/平台汇率的计算逻辑：实时汇率 ×(1±浮动%) | exchange_service, deposit_service |
| `wallet_validator.go` | 余额充足校验、钱包类型合法性校验、biz_type 与 wallet_type 匹配校验 | account_service |
| `format.go` | 金额展示格式化：按币种配置输出千分位+小数位（VND → `1.000.000`，USDT → `0.00000001`） | record_service, 余额查询 |
| `rpc_finance.go` | 调用 ser-finance RPC 的封装：GetDepositMethods / CreateDepositOrder / CreateWithdrawOrder / GetOrderStatus | deposit_service, withdraw_service |
| `rpc_kyc.go` | 调用 ser-kyc RPC 的封装：GetKycStatus / GetKycName | withdraw_service |
| `rpc_user.go` | 调用 ser-user RPC 的封装：GetUserInfo（查用户基本信息） | 需要用户信息的 service |

**设计参照**：ser-item/internal/domain/ 模式——将 RPC 调用、校验逻辑、数据转换从 Service 中抽离，保持 Service 层专注于业务编排。

### 3.9 internal/rep/ — 手写 Repository

> gorm_gen/repo 提供标准 CRUD，但资金操作需要事务+行锁等复杂逻辑，必须手写。

| 文件 | 职责 | 关键操作 |
|------|------|---------|
| `currency_rep.go` | 币种查询、汇率批量更新、偏差判断 | 币种列表查询、汇率批量 Upsert、阈值判断触发平台汇率更新 |
| `account_rep.go` | **最核心** — 余额事务操作 | `SELECT ... FOR UPDATE` 锁行 → 余额校验 → UPDATE 余额+version → INSERT 流水，全在同一 DB 事务内 |
| `deposit_rep.go` | 充值订单查询、重复拦截查询 | 按 user_id+currency+amount+时间窗口查重复；充值订单分页 |
| `withdraw_rep.go` | 提现订单、限额累计、来源判定 | 查当日提现累计金额；查充值来源（判定可用提现方式）；提现订单分页 |
| `exchange_rep.go` | 兑换双钱包原子事务 | 同一事务内 FOR UPDATE 锁两个钱包（有序加锁防死锁）→ 扣源+加目标 → 写两条流水 → 写兑换订单 |
| `record_rep.go` | 四类记录分页查询 | 充值/提现/兑换/奖励记录，按 user_id + currency_code + create_at 筛选分页 |
| `withdraw_account_rep.go` | 提现账户管理 | 查用户提现账户列表、新增、编辑、删除（若支持） |

**构造函数模式**（与 ser-item/internal/rep/ 一致）：

```go
func NewAccountRepo() *AccountRepo {
    db := cfg.InitMySQL()
    query.SetDefault(db)
    return &AccountRepo{
        db:    db,
        redis: cfg.InitRedis(),
    }
}
```

### 3.10 internal/ser/ — Service 层（核心）

> 这一层是**全部业务逻辑的唯一所在地**。handler.go 不做逻辑，rep/ 不做业务判断，所有编排在这里。

| 文件 | 职责 | 对应的 IDL 方法群 | 依赖 |
|------|------|-----------------|------|
| `currency_service.go` | 币种 CRUD + 汇率管理 + 基准币种设置 | B 端：CurrencyList/Add/Edit/Status, RateQuery, RateLogPage, BaseCurrencySet/Get | currency_rep, cache |
| `rate_sync_service.go` | 三方汇率定时拉取 + 阈值判断 + 平台汇率自动更新 | 无直接 IDL 方法（cron 内部触发） | currency_rep, cache, HTTP client |
| `account_service.go` | **最核心** — 余额原子操作 | Credit / Debit / Freeze / Unfreeze / DebitFrozen / GetBalance / GetWalletInfo | account_rep, domain/wallet_validator, cache/dedup |
| `deposit_service.go` | 充值业务：选充值方式 → 创建订单 → 查订单状态 | C 端：DepositMethods, CreateDeposit, DepositCallback, DepositStatus | deposit_rep, domain/rpc_finance, account_service |
| `withdraw_service.go` | 提现业务：KYC 校验 → 冻结 → 创建订单 | C 端：WithdrawMethods, CreateWithdraw, WithdrawAccountList/Add | withdraw_rep, domain/rpc_finance, domain/rpc_kyc, account_service |
| `exchange_service.go` | 兑换业务：预览汇率 → 确认兑换（原子双钱包操作） | C 端：ExchangePreview, ExchangeConfirm | exchange_rep, domain/rate_calculator, cache |
| `record_service.go` | 四类记录查询（充值/提现/兑换/奖励） | C 端：DepositRecordPage/Detail, WithdrawRecordPage/Detail, ... | record_rep, domain/format |
| `bet_service.go` | 投注优先级扣款 + 按比例返奖 | RPC：BetDebit, BetCredit | account_service（组合调用） |
| `wallet_rpc_service.go` | 对外 RPC 统一入口（供 ser-finance 等调用） | RPC：WalletCredit, WalletFreeze, WalletUnfreeze, WalletDebit, ManualCredit/Debit, GetWalletInfo | account_service（包装转发） |
| `enum_service.go` | 枚举查询 | GetEnumOptions | enum/ |

**Service 之间的调用关系**：

```
                    ┌─────────────────────────┐
                    │    account_service.go   │  ← 唯一能改余额的地方
                    │  Credit/Debit/Freeze/   │
                    │  Unfreeze/DebitFrozen    │
                    └──────────┬──────────────┘
                               │ 被以下 Service 调用
            ┌──────────────────┼──────────────────┐
            │                  │                  │
   deposit_service     withdraw_service    exchange_service
   充值成功→Credit     冻结→Freeze           两钱包→Debit+Credit
                       驳回→Unfreeze
                       出款→DebitFrozen
            │                  │
   bet_service         wallet_rpc_service
   扣款→BetDebit       WalletCredit→Credit
   返奖→BetCredit      WalletFreeze→Freeze
```

## 四、common/idl/ser-wallet/ — IDL 文件详解

| 文件 | 定义内容 | 方法/结构体数量 |
|------|---------|---------------|
| `service.thrift` | WalletService 主入口，include 其他 thrift，列出全部 RPC 方法 | ~69 个方法 |
| `currency.thrift` | CurrencyListReq/Resp, CurrencyAddReq, RateQueryResp, BaseCurrencySetReq ... | ~18 个 struct |
| `wallet.thrift` | GetBalanceReq/Resp, WalletInfoResp, WalletItem ... | ~8 个 struct |
| `deposit.thrift` | DepositMethodsResp, CreateDepositReq/Resp, DepositRecordItem ... | ~12 个 struct |
| `withdraw.thrift` | WithdrawMethodsResp, CreateWithdrawReq/Resp, WithdrawRecordItem ... | ~14 个 struct |
| `withdraw_account.thrift` | WithdrawAccountAddReq, WithdrawAccountItem ... | ~6 个 struct |
| `exchange.thrift` | ExchangePreviewReq/Resp, ExchangeConfirmReq/Resp, ExchangeRecordItem ... | ~8 个 struct |
| `record.thrift` | RecordPageReq, RecordDetailReq, RecordItem（通用） ... | ~6 个 struct |
| `wallet_rpc.thrift` | WalletCreditReq/Resp, WalletFreezeReq, BetDebitReq/Resp ... （显式传 userId） | ~16 个 struct |

**wallet_rpc.thrift 单独一个文件的原因**（参照 ser-user/user_rpc.thrift）：
- RPC 接口参数需显式传 `userId`（C 端从 Header 自动获取，RPC 需调用方传入）
- RPC 接口有严格幂等性要求（`orderNo` 字段）
- 其他服务 include 时只需引入这一个文件
