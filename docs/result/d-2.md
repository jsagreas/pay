## 一、多币种钱包模块（你的主要职责）

### 1. 核心架构
- **钱包按币种维度隔离**，每个币种独立余额与账本
- 当前支持币种：**BSB**（平台币，1BSB=0.1USDT）、**VND**、**IDR**、**THB**
- 金额展示：VND/IDR用`.`千分位+整数，THB/BSB用`,`千分位+2位小数
- **不支持跨币种混合展示与操作**

### 2. 钱包体系（来自币种配置文档）
- **资金钱包** = 中心钱包余额 + 奖励钱包余额
- 还有：主播钱包、代理钱包（预留）、场馆钱包（预留）
- 消费优先扣中心钱包，再扣奖励钱包
- 混合投注时需按金额占比分别计返奖

### 3. 充值模块
| 类型 | 方式 | 承载页 |
|------|------|--------|
| 虚拟币 | USDT (TRC20/ERC20/BEP20) | 平台承载 |
| 银行 | 银行转账 | 三方承载 |
| 电子钱包 | QRIS/PromptPay 扫码 | 三方承载 |

**关键规则：**
- 仅BSB充值界面显示汇率换算
- 游客可用USDT充值，登录后解锁更多方式
- **重复转账拦截**：同UserID+同币种+同金额，1-2笔警告可继续，3笔直接拦截
- **额外奖金选择**：活动奖金与充值金额联动，快捷标签带角标，有门槛校验

### 4. 兑换模块（法币→BSB）
- 仅法币钱包有兑换入口（VND/IDR/THB → BSB）
- 汇率统一口径：1 BSB = X 法币
- 赠送奖励（如10%），赠送部分需完成X倍流水后可提现

### 5. 提现模块
**KYC关系：**
- 未认证：弹窗提示，"继续提现"只能USDT，"去认证"解锁全渠道
- 已认证：直接进入提现方式选择

**提现规则：**
- 法币充值 → 法币提现
- USDT充值 → USDT提现
- 法币+USDT混充 → 法币提现
- 法币→BSB → 法币提现

**提现方式按国家：**
- 越南：银行卡（T+1）
- 印尼：GoPay/OVO/DANA/ShopeePay（即时）+ 银行卡
- 泰国：PromptPay（当天）+ 银行卡
- BSB：USDT + 各国银行/电子钱包（带汇率展示）

**账户信息：** 首次填写，平台记录，后续复用；持有人姓名=KYC姓名（不可编辑）

### 6. 记录模块
- 分类：提现/充值/兑换/奖励
- 当前币种=记录币种，切换币种即切换数据
- 详情页含：订单号、金额明细、状态、时间、失败原因

### 7. 非功能性需求
- 账务流水与余额的**强一致性**
- 充值回调**幂等处理**
- KYC强制校验、提现账户白名单、姓名强一致校验
- 大额/异常频率触发人工审核

## 二、币种配置模块（你的主要职责，服务于钱包）

### 1. 币种类型
| 类型 | 示例 |
|------|------|
| 法定货币 | IDR、VND、THB |
| 加密货币 | USDT、BTC、ETH |
| 平台货币 | BSB |

### 2. 基准币种
- 推荐USDT，配置后**不可更改**
- BSB锚定：10 BSB = 1 USDT
- 用途：活动奖金计算、多币种统计、财务报表

### 3. 后台配置页
- 字段：币种代码/名称/类型/国籍/图标/千分位符号/小数点符号/实时汇率/平台汇率/汇率浮动/入款汇率/出款汇率/偏值/状态等
- 编辑弹窗可配：汇率浮动（%）、币种图标（SVG/PNG≤5k）、偏值（%）、启用/禁用
- 禁用币种后C端不可使用/充值/提现

### 4. 汇率规则（核心）
- **实时汇率**：3家三方API取均值
- **平台入款汇率** = 实时汇率 × (1 + 汇率浮动%)
- **平台出款汇率** = 实时汇率 × (1 - 汇率浮动%)
- **偏差值** = |实时汇率 - 平台汇率| / 实时汇率 × 100%
- 偏差值 ≥ 阈值时 → 更新该币种平台汇率 → 生成汇率日志
- 汇率展示：法币保留2位小数，加密货币保留8位小数

## 三、财务管理模块（他人负责，关注依赖关系）

### 关键子模块
1. **通道配置**：代收/代付通道参数管理，轮询规则（成功率/时间/预警系数）
2. **支付配置**：充值方式/提现方式配置（档位、金额范围、关联通道、图标）
3. **充值记录**：含充值流程（轮询匹配通道→创建订单→回传结果→上账）
4. **提现记录**：含提现流程（风控审核→财务审核→出款），订单状态多级（待风控/风控驳回/待财务/财务驳回/待出款/出款失败/出款成功）
5. **人工修正**：充值补单/人工加款/人工减款（双人审核，操作人≠审核人）
6. **数据统计**：通道统计/充值统计/提现统计

## 模块间依赖关系初步判断

| 依赖方向 | 说明 |
|----------|------|
| **钱包 → 币种配置** | 钱包依赖币种配置获取：币种列表、汇率、展示规则 |
| **钱包 → 财务（支付配置）** | 充值/提现方式、档位、金额范围来源于财务的支付配置 |
| **钱包 → 财务（通道）** | 充值创建订单时需轮询匹配通道（财务负责通道配置与轮询策略） |
| **财务 → 钱包** | 充值回调后需调用钱包服务上账/扣款；人工加减款直接操作钱包余额 |
| **财务 → 币种配置** | 统计报表需按基准币种换算；记录中需用平台汇率 |
