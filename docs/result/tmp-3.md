## 三大模块需求理解汇总

### 模块一：多币种钱包（主责）

**钱包体系：**
- 资金钱包按**币种维度隔离**，每个币种一套独立余额与账本
- 当前支持：BSB（平台币，1BSB=0.1USDT）、VND、IDR、THB
- 钱包内部分层：中心钱包、奖励钱包、主播钱包、代理钱包（预留）、场馆钱包（预留）
- 资金钱包余额 = 中心钱包 + 奖励钱包；消费时优先扣中心钱包再扣奖励钱包
- 混合投注时需按金额占比分开计返奖

**充值模块（5个子场景）：**
- USDT充值（平台承载页，TRC20/ERC20/BEP20，链上监听到账）
- 银行转账充值（三方承载页，支付通道回调）
- 扫码充值（QRIS/PromptPay，三方二维码+倒计时）
- 重复转账拦截规则（同用户+同币种+同金额，1-2笔提示可继续，3笔强制拦截）
- 额外奖金选择规则（活动关联充值，金额加成，流水倍数要求）

**兑换模块：** 法币→BSB单向兑换，汇率展示1BSB=X法币，含赠送比例（如10%），赠送部分需完成X倍流水

**提现模块：**
- KYC为可选但解锁高阶渠道的前置；未认证仅限USDT提现
- 提现规则：法币充→法币提，USDT充→USDT提，混合充→法币提
- 银行卡提现（首次填写/后续复用）、电子钱包提现（GoPay/OVO/DANA/PromptPay等）、BSB提现（含汇率换算）
- 账户持有人=KYC姓名（自动填充不可编辑）
- 提现限制：单笔限额、单日限额、手续费、T+N到账

**记录模块：** 提现/充值/兑换/奖励四Tab，按当前币种隔离，不支持跨币种混合；详情以弹窗形式展示

**非功能性：** 对账强一致性、充值回调幂等、KYC强制校验、姓名强一致、大额/异常频率触发人工审核

### 模块二：币种配置（主责，服务于钱包）

**币种管理：**
- 三类币种：法定货币、加密货币、平台货币
- 币种数据由开发导入，后台可编辑图标、汇率浮动%、阈值%、启用/禁用状态
- 基准币种（推荐USDT）只能设置一次不可更改

**汇率体系（核心）：**
- 实时汇率：3家三方API取平均值
- 平台汇率：等于实时汇率（当偏差值>阈值时更新）
- 入款汇率 = 实时汇率 × (1 + 浮动%)
- 出款汇率 = 实时汇率 × (1 - 浮动%)
- 偏差值 = |实时汇率 - 平台汇率| / 实时汇率 × 100%
- 偏差值>阈值 → 更新平台汇率+入款汇率+出款汇率 → 生成汇率日志
- 法币展示小数点后2位，加密货币小数点后8位

**汇率日志：** 在操作日志页新增"汇率日志"Tab，记录每次汇率变更

### 模块三：财务管理（参考，他人负责）

**8个子模块：**
1. **通道配置** — 代收/代付通道管理+轮询规则（成功率/时间/预警三系数综合评分）
2. **支付配置** — 充值方式/提现方式配置（档位、范围、关联通道、费率）
3. **充值记录** — 充值订单管理+流程（含基准币种折算）
4. **提现记录** — 提现订单管理+审核流程（风控审核→财务审核→出款）+锁定/解锁
5. **人工修正** — 充值补单（B+前缀）、人工加款（A+前缀）、人工减款（M+前缀），均需双人审核
6. **通道统计** — 代收/代付通道维度统计
7. **充值统计** — 按日+币种维度统计
8. **提现统计** — 按日+币种维度统计（含风控/财务审核维度）

## 模块间依赖关系（初步识别）

| 依赖方向 | 说明 |
|---------|------|
| 钱包 → 币种配置 | 钱包的币种列表、汇率（入款/出款/平台）全部来自币种配置 |
| 钱包 → 财务(支付配置) | C端充值方式列表、档位、关联通道由财务模块的"支付配置"决定 |
| 钱包 → 财务(通道配置) | 充值时的通道匹配由财务模块的"轮询规则"决定 |
| 财务(充值记录) → 钱包 | 充值成功后需调用钱包接口给用户上账 |
| 财务(提现记录) → 钱包 | 提现成功需扣减钱包余额，提现驳回需退回余额 |
| 财务(人工修正) → 钱包 | 补单/加款/减款直接操作用户钱包余额（按钱包类型） |
| 钱包提现 → KYC服务 | 提现触发KYC认证校验 |

## 详细需求文档解析

### 一、多币种钱包需求文档（完整提取）

#### 1. 需求背景

随着平台支持 **直播 + 游戏 + 打赏 + 多国家市场**，资金体系逐步复杂化，需建立一套 **清晰、安全、可扩展的多币种钱包系统**，以支撑以下核心能力：

- 多币种充值与支付
- 游戏投注与结算
- 礼物打赏与主播收益
- 提现与合规风控（KYC）
- 资金流水与对账追溯

钱包系统作为平台 **资金与虚拟资产的核心中枢**，需做到：

- 用户易理解
- 技术易实现
- 风控可控
- 后期可扩展

#### 2. 需求目标

1. 支持多币种钱包体系（法币 / BSB币）
2. 明确资产分层，避免资金逻辑混乱
3. 支持不同币种对应不同充值 / 提现方式
4. 提供完整、可追溯的资金记录
5. 确保提现合规（KYC + 风控）

#### 3. 适用范围

- **客户端：** iOS / Android / H5
- **用户角色：** 普通用户、主播（含签约 / 非签约）

#### 4. 钱包总体设计

**4.1 钱包币种结构**
- 钱包按 **币种维度隔离**，每个币种一套独立余额与账本
- 当前支持币种：BSB（锚定汇率：1BSB=0.1USDT）、VND、IDR、THB
- 金额展示规则：

| 币种 | 千分位规则 | 小数点规则 |
|------|----------|----------|
| VND  | .        | 整数     |
| IDR  | .        | 整数     |
| THB  | ,        | 2位小数   |
| BSB  | ,        | 2位小数   |

**4.2 币种切换规则**
- 钱包首页可切换当前币种
- **当前选中币种 = 所有操作的唯一数据源：** 充值、提现、兑换、交易记录
- 切换币种即切换：余额、充值/支付方式、记录列表数据
- 不支持跨币种混合展示与操作

**4.3 页面交互**
- 点击切换：跳转选择币种界面
- 点击充值：温馨提示弹窗
- 继续充值：充值界面（仅支持USDT充值，不支持法币充值）
- 去登陆：跳转登陆界面（登陆后返回原界面，再点击充值，展示所有充值方式）
- 点击记录：跳转该币种记录界面

#### 5. 充值模块 PRD

**5.1 充值方式总览**

| 类型     | 方式                    | 是否平台承载页 | 状态回调           | 备注                    |
|----------|-----------------------|------------|----------------|-----------------------|
| 虚拟币   | USDT（TRC20 / ERC20） | 是         | 第三方支付通道回调 | 链上转账               |
| 银行     | 银行转账               | 否         | 第三方支付通道回调 | 第三方通道               |
| 电子钱包 | QRIS / PromptPay 扫码  | 否         | 第三方支付通道回调 | 第三方通道二维码 + 倒计时 |

规则说明：
1. 每个币种进来有默认充值方式及默认最小档位快捷金额
2. 充值金额输入框支持手动填写（手动填写数额不得小于最低档位）
3. 点击去充值后生成订单匹配通道
4. 只有BSB充值金额输入框下有对应汇率展示及兑换所得的金额

**5.2 USDT 充值**

- 入口规则：仅在支持USDT支付方式的币种下可见；游客登录状态也支持USDT充值
- 金额规则：充值单位为钱包币种单位，系统根据汇率自动换算USDT
- USDT支付页：网络选择TRC20（默认），可扩展ERC20/BEP20；展示网络类型、收款地址（可复制）、二维码
- 到账机制：通过区块链监听服务确认到账，达到确认数后自动上账

**5.3 银行转账充值**

- 入口规则：根据当前币种展示对应国家银行转账
- 流程：用户选择充值金额 → 点击"立即充值" → 平台向支付通道创建订单 → 获取并展示转账信息（三方承载页）→ 支付状态回传平台 → 平台更新订单状态
- 回调说明："查看详情"用于查看订单状态；"继续转账程序"用于返回三方台面

**5.4 QRIS / PromptPay 扫码充值**

- 入口规则：印尼/泰国法币钱包下展示，默认推荐方式
- 承载页（三方）：展示二维码、倒计时、支付方式、金额（可复制）、备注
- 状态规则：支付状态回传平台，平台更新订单状态

**5.5 重复转账拦截规则**

- 识别维度：同一UserID + 同一币种 + 同一充值金额
- 状态定义：订单处于"待支付"或"进行中"
- 规则适用：银行转账、电子钱包支付（不适用于USDT链上充值）

| 待支付订单数 | 弹窗文案 | 按钮1 | 按钮2 | 交互 |
|-----------|---------|------|------|------|
| 1~2 笔 | 您有N笔同等金额交易正在进行中，建议您先处理已有订单，是否确认继续创建新充值？ | 查看详情 | 继续充值 | 点击后续流程不变 |
| 3 笔（上限）| 您有3笔同等金额的交易正在进行中，为确认先处理已有订单再继续新交易。 | 查看详情 | 确认 | 强制拦截，不触发订单创建 |

**5.6 额外奖金选择规则**

- 组件位置：位于"余额展示区"下方、"支付方式"上方
- 交互逻辑：默认状态显示"无奖金"或当前最优匹配活动
- 点击弹出"请选择您的奖金"半屏浮窗
- 浮窗内容：列出所有可用奖金活动，提供"无奖金"选项，底部强制提示流水要求
- 奖金与充值金额联动：快捷金额标签右上角带奖金比例角标（如+50%），输入框右侧展示"额外+XXX"
- 门槛校验：手动输入金额低于最低充值门槛时警告提示
- 状态互斥：同一笔充值订单仅关联一个奖金活动

**5.7 页面交互**
- 左上角：跳转返回个人中心
- 右上角：跳转客服中心
- 点击切换：币种切换（余额跟随币种变化，个人中心币种不受影响）
- 点击提现：弹出KYC认证温馨提示
- 点击活动奖金充值：弹出"请选择您的奖金"半屏浮窗
- 切换其他支付方式：底下的档位根据支付方式会发生变化
- 点击去充值：跳转三方承载页/USDT支付页
- 查看详情：跳转充值记录界面
- 继续转账程序：返回三方承载页

#### 6. 兑换模块（法币 → BSB）

**6.1 兑换范围**
- 支持：VND / IDR / THB → BSB（BSB充值界面无兑换icon）
- 仅限当前币种余额兑换

**6.2 汇率规则**
- 统一展示口径：**1 BSB = X 法币**
- 示例：1 BSB = d2,620 / 1 BSB = Rp 1,530 / 1 BSB = B3.50

**6.3 兑换计算**
- 兑换金额：输入当前法币金额（超出余额提示：已达当前可兑换上限）
- 兑换获得 BSBxxxx.xx
- 赠送奖励（如 10%）
- 实际到账 BSB = 兑换获得 + 赠送

**6.4 赠送规则**
- 赠送部分需完成 **X 倍流水** 后可提现
- 在页面明确提示

**6.5 页面交互**
- 左上角：返回该币种充值中心界面
- 点击记录：跳转记录界面
- 点击兑换：3S内兑换成功提示

#### 7. 提现模块

**7.1 提现总原则**

1. KYC与提现关系：KYC为可选非强制（针对USDT），但作为解锁高阶提现渠道的必要前提。
   - 未认证状态：点击提现触发弹窗，提供"去认证"和"继续提现"两个入口
   - 已认证状态：点击提现直接进入提现方式选择页

| 选项 | 触发动作 | 最终权限 |
|------|---------|---------|
| "去认证" | 进入KYC认证页面 | 审核通过后，解锁法币提现/电子钱包/USDT等全渠道 |
| "继续提现" | 跳过KYC认证 | 仅限使用USDT提现 |

2. 提现规则：
   - 法币充值 → 法币提现
   - USDT充值 → USDT提现
   - 法币+USDT充值 → 法币提现
   - 法币 → BSB → 法币提现

3. 首次提现需输入账户信息
4. 提现账户信息平台记录
5. 后续提现无需重复填写（可更改提现信息）
6. **账户持有人姓名 = KYC认证姓名**（提现页自动填充已实名的姓名且不可编辑）
7. 账户信息未填写完成 → 确认提现icon置灰；必填项全部填写完成 → icon高亮
8. 确认提现校验提示：输入金额超出每日提现限额/未达单笔最低时弹窗提示

**7.2 银行卡提现**
- 首次提现：填写银行名称、银行账号、账户持有人（置灰默认kyc名字）、开户网点、提现金额
- 非首次提现：自动带出账户信息（只读），仅输入提现金额

**7.3 电子钱包提现**
- 支持：GoPay / OVO / DANA / PromptPay 等
- 首次提现：钱包类型、手机号、账户持有人（置灰默认kyc名字）、提现金额
- 后续提现复用（只读）

**7.4 BSB提现**
- 支持：USDT/各大银行/GoPay/OVO/DANA/PromptPay等
- 只有BSB提现金额输入框下有对应**汇率展示**
- 后续提现复用（只读）

**7.5 提现限制（后台配置）**
- 单笔限额、单日限额、手续费规则、T+N到账时间

**7.6 页面交互**
- 左上角：返回该币种充值中心界面
- 点击记录：跳转记录界面
- 选择提现方式：跳转该方式提现信息输入页面
- 点击确认提现：弹出"提现状态"半屏浮窗
- 点击查看详情：跳转提现记录

#### 8. 记录模块

**8.1 记录分类：** 提现、充值、兑换、奖励

**8.2 核心规则**
- 钱包当前币种 = 记录当前币种
- 切换币种即切换整套记录数据
- 不支持跨币种混合记录

**8.3 列表页**
- 提现字段：金额、方式、时间、状态（颜色区分：已完成/审核中/失败）、操作
- 充值字段：金额、方式、时间、状态（颜色区分：已完成/审核中/失败）、操作
- 兑换字段：兑换金额、赠送x%、实际到账、时间
- 奖励字段：奖励金额、奖励类别、时间

**8.4 详情页**
- 所有金额币种必须与列表页币种一致
- 展示：订单号、金额明细、状态、时间、方式、失败原因（如有）
  - 充值失败原因：支付超时、支付异常、卡片问题
  - 提现失败原因：信息有误、系统维护、额度受限

**8.5 页面交互**
- 左上角：返回个人中心界面
- 右上角：币种切换
- 顶部菜单栏：提现/充值/兑换/奖励记录相互切换
- 详情：弹出查看提现、充值详情弹窗
- 更换其他提现方式：跳转提现界面
- 更换其他充值方式：跳转充值界面

#### 9. 非功能性需求

- **对账一致性：** 明确"账务流水"与"账户余额"的强一致性校验机制
- **幂等性：** 充值回调必须做幂等处理，防止重复入账

#### 10. 风控与异常处理

- KYC强制校验
- 提现账户白名单
- 姓名强一致校验
- 不支持用户自行修改提现账户
- 异常情况需人工介入
- 大额/异常频率触发人工审核

### 二、多币种钱包产品原型（完整描述）

#### 1-三方充值页面

两个并排手机界面，QRIS支付页面：
- 左侧（正常状态）：标题"QRIS支付"，倒计时"04:56"，二维码，支付方式QRIS，金额RP 3,000,000（可COPY），备注UserID+时间戳（可COPY），操作说明三步

### 三、币种配置需求文档（完整提取）

#### 1. 币种配置说明

**文档背景：** 钱包体系作为平台的核心基础能力之一，用于承载用户在平台内的资金流转与消费行为，需同时满足娱乐消费与igaming投注两类业务场景。

**产品目标：** 本体系通过清晰的钱包类型划分与严格的资金流向规则，确保用户体验一致，资金安全可控，并支持后续业务的财务、风控能力的扩展。

**文档规范：** 绿字=解释说明，红字=特别注意，灰字=当前版本暂不制作，删除线=删除移除

#### 2. 资金钱包体系

**2.1 资金钱包定义**
- 资金钱包用于承载用户在平台内的真实资金，用于igaming投注、结算、兑换钻石及提现等操作，是风控与财务的核心模块
- 资金钱包支持平台币、多法币、多虚拟币，C端钱包中各币种金额相互独立
- 平台币暂命名为：BSB

**钱包结构树：**
- 中心钱包 → 完成稽核流水可提现
- 奖励钱包 → 完成稽核流水进入中心钱包
- 场馆钱包(预留)
- 代理钱包 → 可直接提现，可转入中心钱包（转入后需稽核）
- 主播钱包 → 可直接提现，可转入中心钱包

**使用场景：** 娱乐场消费（刷礼物、购买虚拟道具等）、投注（电子/真人视讯/彩票/体育）、提现（法币充值可提法币，虚拟币充值可提虚拟币；混合充值只可提法币）

**获得途径：** 充值（直充法币或虚拟币）、投注返奖、活动/任务返奖

**资金流向：**
- 活动/任务返奖 → 奖励钱包
- 投注返奖 → 中心钱包
- 充值 → 中心钱包
- 直播礼物收益 → 主播钱包
- 代理佣金 → 代理钱包(预留)

**各钱包说明：**
- **中心钱包：** 用户主资金钱包，承担充值、投注扣减、投注返奖、提现功能
- **奖励钱包：** 活动/任务奖金暂存钱包，需完成稽核流水后可转入中心钱包
- **主播钱包：** 承接直播/连麦/短视频等内容场景分成收益，支持提现和转入中心钱包
- **代理钱包(预留)：** 代理专属资金钱包，承接下级分佣收益
- **场馆钱包(预留)：** 进入igaming场馆时的临时资金钱包

**余额规则：**
- 资金钱包余额 = 中心钱包余额 + 奖励钱包余额
- 消费时优先扣除中心钱包余额，再扣奖励钱包余额
- 混合投注时需根据投注金额占比分开计返奖

#### 4. 汇率规则说明

**4.1 汇率应用场景**
- 充值：跨币种充值时使用平台入款汇率
- 提现：跨币种提现时使用平台出款汇率
- 投注：游戏不支持当前钱包币种时根据入款汇率转换，退出时使用该汇率转换回
- 统计：充值/提现/账变等统计根据平台汇率转化为基准货币

**4.2 汇率规则**
- **实时汇率：** 通过3家三方API取平均值，可约定刷新频率
- **平台汇率：** 自动刷新的实时汇率，初始为空直接取实时汇率值
- **入款汇率** = 实时汇率 × (1 + 汇率浮动%)
- **出款汇率** = 实时汇率 × (1 - 汇率浮动%)
- **偏差值** = (实时汇率 - 平台汇率) / 实时汇率 × 100
- **阈值：** 可配置，偏差值>阈值%时更新平台汇率并生成日志

**汇率更新流程：**
1. 实时汇率更新 → 是否存在平台汇率？
2. 否 → 根据实时汇率计算平台汇率
3. 是 → 偏差值是否>=阈值%？
4. 是 → 根据实时汇率计算新的平台汇率
5. 否 → 当前平台汇率保持不变

**4.3 汇率展示**

| 货币类型 | C端展示汇率 | 系统侧汇率 |
|---------|-----------|-----------|
| 法定货币 | 保留小数点后2位 | 保留小数点后2位，不足补0，超过四舍五入 |
| 加密货币 | 保留小数点后8位 | 保留小数点后8位，不足补0，超过四舍五入 |

**4.4 汇率日志**

在操作日志页增加"汇率日志"页签，字段：
1. 序号
2. 日志编号 - 唯一编码
3. 更新时间
4. 币种代码
5. 实时汇率
6. 平台汇率
7. 入款汇率
8. 出款汇率

- 管理后台可设置站点时区，查询时需还原回UTC+0
- 代收/代付通道统计，按日期+时间范围+通道+币种维度
- 汇总卡片：订单总数/成功订单总数/成功率/平均入款时间/订单总金额/成功订单金额/净入账金额/手续费
- 金额汇总以基准币种计算
- 支持导出

**3.2 充值统计**
- 按日期+币种维度
- 汇总卡片：订单总数/成功订单总数/充值人数/成功充值人数/成功率/平均入款时间/手续费/人均充值金额
- 表格含待支付订单/失败订单统计

**3.3 提现统计**
- 按日期+币种维度
- 汇总卡片：订单总数/成功订单总数/提现人数/成功提现人数/成功率/平均出款时间/平台手续费/通道手续费/待出款订单金额/出款失败订单金额
- 表格含风控审核/财务审核各阶段统计

### 六、财务管理产品原型（完整描述）

#### 通道配置系列
- 通道配置-1：代收通道列表页（筛选+列表+分页），示例数据VND/IDR，支付类型唤醒/原生/混合/扫码

#### 提现记录系列
