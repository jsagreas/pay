## 三大模块需求理解总结

### 一、币种配置模块（你负责 — 基础设施层）

**核心定位**：为整个钱包和财务体系提供币种定义、汇率计算、钱包分层结构的基础配置能力。

**资金钱包体系（5种子钱包）**：
| 子钱包 | 资金来源 | 使用/流向 | 状态 |
|--------|---------|----------|------|
| **中心钱包** | 充值 | 投注、娱乐消费、提现（需完成稽核） | 核心 |
| **奖励钱包** | 活动/任务返奖、投注返奖 | 完成流水稽核后→转入中心钱包 | 核心 |
| **主播钱包** | 礼物打赏、短视频收益 | 可直接提现/可转入中心钱包（转入后需稽核） | 核心 |
| **代理钱包** | 代理佣金 | 可直接提现/可转入中心钱包 | **预留** |
| **场馆钱包** | 中心+奖励钱包转入 | 进入游戏场馆时临时使用，退出归还 | **预留** |

**关键余额公式**：资金钱包余额 = 中心钱包余额 + 奖励钱包余额
**消费优先级**：先扣中心钱包，再扣奖励钱包
**混合投注返奖**：按投注金额比例分别归入对应钱包

**币种类型**：法定货币（VND/IDR/THB）、加密货币（USDT/BTC/ETH）、平台货币（BSB）

**基准币种**：USDT，只能设置一次不可修改。BSB 锚定 USDT（10 BSB = 1 USDT）

**汇率体系**：
- 实时汇率 = 3个第三方API取平均值
- 平台入款汇率 = 实时汇率 × (1 + 汇率浮动%)
- 平台出款汇率 = 实时汇率 × (1 - 汇率浮动%)
- 偏差值 = |实时汇率 - 平台汇率| / 实时汇率 × 100%
- 偏差值 ≥ 阈值% → 更新平台汇率（入款和出款任一超阈值都触发）

**精度规则**：法币保留2位小数，加密货币保留8位小数（四舍五入、不足补0）

**后台配置页**：20个字段的表格（含实时汇率倒计时、编辑弹窗上传币种图标SVG/PNG≤5KB、启禁用状态等）+ 汇率日志Tab

### 二、多币种钱包模块（你负责 — 核心业务层）

**支持币种**：BSB、VND、IDR、THB（按币种维度隔离，不支持跨币种混合展示和操作）

**金额显示规则**：
| 币种 | 千分位 | 小数 |
|------|--------|------|
| VND/IDR | `.` | 整数（无小数） |
| THB/BSB | `,` | 2位小数 |

**充值模块**（3种方式）：
- **USDT**：平台承载页，TRC20(默认)/ERC20/BEP20，链上确认到账，游客也可用
- **银行转账**：第三方托管页，回调更新状态
- **电子钱包扫码**（QRIS/PromptPay）：第三方托管页，倒计时+二维码

**重复转账拦截**：同UserID+同币种+同金额，1-2笔软拦截（可继续），≥3笔硬拦截（禁止创建）

**额外奖金**：单笔订单互斥，有最低充值门槛校验，奖金部分需完成N倍流水

**兑换模块**：仅法币→BSB 单向兑换，赠送部分需完成流水后可提现

**提现模块**：
- KYC 可选但影响可用渠道：未认证仅USDT提现，认证后全渠道
- **提现币种规则**：法币充值→法币提现，USDT充值→USDT提现，混合充值→法币提现，法币→BSB→法币提现
- 首次需填写账户信息，后续复用（不可自行修改）
- 账户持有人 = KYC实名（灰显不可编辑，忽略大小写和多余空格比对）
- 后台可配：单笔限额、单日限额、手续费率、T+N到账时间

**记录模块**：4个Tab（提现/充值/兑换/奖励），状态色标（已完成/审核中-橙/失败-红），失败有原因和"更换方式"链接

**非功能性**：账务流水与余额强一致性校验、充值回调幂等、大额/异常频率触发人工审核

### 三、财务管理模块（他人负责 — 辅助理解依赖）

**功能清单**：通道配置（代收/代付）、支付配置（充值/提现方式）、充值记录、提现记录、人工修正（补单/加款/减款）、数据统计

**与钱包模块的关键依赖关系**：

| 方向 | 操作 | 钱包侧影响 |
|------|------|-----------|
| 财务→钱包 | 充值成功 | 订单金额入中心钱包，奖金入奖励钱包 |
| 财务→钱包 | 创建提现订单 | **冻结**用户钱包对应金额 |
| 财务→钱包 | 提现风控/财务驳回 | **解冻**归还冻结金额 |
| 财务→钱包 | 提现出款失败 | **归还**资金到钱包 |
| 财务→钱包 | 提现出款成功 | **最终扣除**冻结资金 |
| 财务→钱包 | 补单审核通过 | 补充金额+补充奖金入账 |
| 财务→钱包 | 人工加款审核通过 | 按4种子钱包分别加款 |
| 财务→钱包 | 人工减款审核通过 | 按4种子钱包分别减款（余额不足则实际扣款=当前余额） |
| 钱包→财务 | 读取余额 | 人工加/减款弹窗需展示当前各子钱包余额 |

**订单号规范**：C+16位（充值）、T+16位（提现）、B+16位（补单）、A+16位（加款）、M+16位（减款）

**提现流程**：用户发起 → 风控审核 → 财务审核 → 出款（通道代付）→ 成功/失败回调

**通道轮询**：按权重+轮询系数（成功率/时间/预警）选择代收/代付通道
