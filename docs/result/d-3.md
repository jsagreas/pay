## 一、币种配置模块（你负责，服务于钱包）

### 1.1 核心定位
币种配置是整个资金体系的**基础底座**，为钱包、充值、提现、兑换、投注、统计等所有涉及资金的场景提供币种定义、汇率计算的基础能力。

### 1.2 资金钱包体系（重要架构）

**钱包结构（5种）：**

| 钱包类型 | 说明 | 资金来源 | 提现规则 |
|---------|------|---------|---------|
| **中心钱包** | 用户主钱包 | 充值、稽核完成的奖励金、主播/代理转入 | 完成稽核流水后可提现 |
| **奖励钱包** | 活动/任务奖金暂存 | 活动/任务返奖、投注返奖 | 需完成稽核流水后转入中心钱包 |
| **主播钱包** | 主播收益 | 礼物打赏、主播活动奖励 | 可直接提现，也可转入中心钱包 |
| **代理钱包(预留)** | 代理佣金 | 代理分成 | 可直接提现，也可转入中心钱包 |
| **场馆钱包(预留)** | 游戏临时钱包 | 中心+奖励钱包转入 | 退出场馆时转回 |

**关键规则：**
- 资金钱包余额 = 中心钱包余额 + 奖励钱包余额
- 消费优先扣除中心钱包，再扣奖励钱包
- 混合投注时按金额占比分别计返奖

### 1.3 币种类型划分

| 类型 | 示例 | 说明 |
|-----|------|------|
| 法定货币 | VND、IDR、THB | 对应国家/地区 |
| 加密货币 | USDT、BTC、ETH | 区块链数字资产 |
| 平台货币 | BSB | 平台专属，10BSB=1USDT，锚定USDT |

### 1.4 基准币种
- 推荐USDT，**配置后不可更改**
- 用于：活动奖金计算、多币种统计、财务对账报表
- 所有币种金额在账务层需换算为基准币种再统计

### 1.5 汇率体系（核心）

| 概念 | 说明 |
|------|------|
| **实时汇率** | 3家三方API取平均值 |
| **平台汇率** | 初始=实时汇率，后续根据阈值判断是否更新 |
| **汇率浮动%** | 后台可配置，每币种独立 |
| **入款汇率** | 实时汇率 × (1 + 浮动%) |
| **出款汇率** | 实时汇率 × (1 - 浮动%) |
| **偏差值** | \|实时 - 平台\| / 实时 × 100 |
| **阈值** | 可配置，偏差值 >= 阈值时更新平台汇率 |

**汇率应用场景：**
- 充值：跨币种充值用**入款汇率**
- 提现：跨币种提现用**出款汇率**
- 投注：进入游戏时入款汇率转换，记录该汇率，退出时原汇率转回
- 统计：所有报表用平台汇率转基准币种

**展示规则：**
- 法币：小数点后2位
- 加密货币：小数点后8位

### 1.6 后台功能
- 币种列表（CRUD、状态开关、图标上传SVG/PNG≤5k）
- 编辑弹窗（汇率浮动%、阈值%、状态启用/禁用）
- 汇率日志（平台汇率变更时自动生成）
- 操作日志

## 二、多币种钱包模块（你的核心模块）

### 2.1 钱包总体设计
- 按**币种维度隔离**，每个币种独立余额和账本
- 当前支持：BSB、VND、IDR、THB
- 金额展示规则：VND/IDR用`.`做千分位且为整数；THB/BSB用`,`做千分位且保留2位小数

### 2.2 币种切换规则
- 切换币种 = 切换**所有操作的数据源**（余额、充值方式、提现方式、记录）
- **不支持跨币种混合展示与操作**

### 2.3 充值模块

**三种充值方式：**

| 方式 | 平台承载页 | 回调 | 适用场景 |
|------|----------|------|---------|
| USDT（TRC20/ERC20/BEP20） | 是（平台页） | 区块链监听 | 支持USDT的币种 |
| 银行转账 | 否（三方页） | 三方回调 | 对应国家银行 |
| 电子钱包扫码（QRIS/PromptPay） | 否（三方页） | 三方回调 | 印尼/泰国 |

**关键业务规则：**
- 只有BSB充值时下方展示汇率换算
- 重复转账拦截：同UserID+同币种+同金额，1-2笔警告可继续，3笔强制拦截
- 额外奖金选择：充值时可选奖金活动，有最低充值门槛，需完成X倍流水
- 游客可USDT充值，登录后解锁全部方式

### 2.4 兑换模块（法币→BSB）
- 仅支持VND/IDR/THB → BSB（单向）
- 汇率口径：1 BSB = X 法币
- 赠送规则：赠送10%（赠送部分需完成X倍流水后可提现）
- 实际到账 = 兑换获得 + 赠送

### 2.5 提现模块

**KYC与提现关系：**
- 未认证：弹窗提示，可选"去认证"或"继续提现"（仅USDT）
- 已认证：直接进入提现方式选择

**提现规则：**
- 法币充值 → 法币提现
- USDT充值 → USDT提现
- 法币+USDT混合充值 → 只能法币提现
- 法币→BSB→ 法币提现

**提现方式：** 银行卡（T+1）、电子钱包（即时）、USDT（即时）
- 首次提现需填写账户信息，后续复用
- 账户持有人 = KYC认证姓名（自动填充不可编辑）
- 单笔限额、单日限额、手续费均后台可配
- BSB提现时展示汇率换算

### 2.6 记录模块
- 四个Tab：提现、充值、兑换、奖励
- 核心规则：当前币种 = 记录币种，不支持跨币种混合
- 详情页展示订单号、金额明细、状态、时间、方式、失败原因

### 2.7 非功能性需求
- 对账一致性：账务流水与账户余额强一致
- 幂等性：充值回调幂等处理，防重复入账
- 风控：KYC校验、提现账户白名单、姓名强一致、大额/异常频率人工审核

## 三、财务管理模块（他人负责，你需理解依赖关系）

### 3.1 模块结构

| 子模块 | 核心功能 |
|--------|---------|
| **通道配置** | 代收/代付通道管理、轮询规则配置（配比+优先级） |
| **支付配置** | 充值方式/提现方式管理（关联通道、金额范围、图标、排序） |
| **充值记录** | 充值订单查询/导出、完整充值流程（客户端→服务端→通道） |
| **提现记录** | 提现订单查询/导出、三级审核（风控→财务→出款）、锁定/解锁 |
| **人工修正** | 充值补单、人工加款（4种钱包分别操作）、人工减款 |
| **通道统计** | 代收/代付通道成功率、金额、笔数统计 |
| **充值/提现统计** | 按日期+币种维度统计 |

### 3.2 与钱包模块的关键依赖点

**钱包 → 财务（你调用他）：**
1. **充值下单**：钱包发起充值时，需要调用财务的通道分配/轮询逻辑，获取支付通道信息
2. **提现下单**：钱包发起提现时，需要提交到财务的审核流程（风控→财务→出款）
3. **支付方式/提现方式查询**：充值/提现页面展示的方式列表来自财务的支付配置

**财务 → 钱包（他调用你）：**
1. **充值成功回调**：通道回调充值成功后，财务需调用钱包的**上分/入账接口**
2. **提现成功回调**：出款成功后，财务需确认钱包已**扣款**
3. **人工加款/减款**：财务审核通过后，需调用钱包的**加款/减款接口**（涉及中心钱包、奖励钱包、代理钱包、主播钱包4种）
4. **充值补单**：补单审核通过后，需调用钱包入账（含稽核倍数/流水设置）

## 四、理解总结

1. **核心交付**：多币种钱包的C端功能（充值、提现、兑换、记录）+ 币种配置的B端管理功能（币种CRUD、汇率体系）
2. **币种配置是底层基建**：为钱包提供币种定义、汇率计算能力，所有涉及跨币种计算的场景都依赖它
3. **与财务模块的边界**：钱包负责钱包内的资金账户管理（余额、入账、扣款），财务负责通道对接、订单流转、审核流程、统计报表
4. **需要暴露的RPC接口方向**：主要是入账（充值成功）、扣款（提现扣款）、加款/减款（人工修正）、余额查询、钱包信息查询等
