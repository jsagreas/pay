# 第一部分：多币种钱包（主责模块）

## 一、需求文档分析

### 1. 需求背景

随着平台支持**直播 + 游戏 + 打赏 + 多国家市场**，资金体系逐步复杂化，需建立一套**清晰、安全、可扩展的多币种钱包系统**，以支撑以下核心能力：

- 多币种充值与支付
- 游戏投注与结算
- 礼物打赏与主播收益
- 提现与合规风控（KYC）
- 资金流水与对账追溯

钱包系统作为平台**资金与虚拟资产的核心中枢**，需做到：
- 用户易理解
- 技术易实现
- 风控可控
- 后期可扩展

### 2. 需求目标

1. 支持多币种钱包体系（法币 / BSB币）
2. 明确资产分层，避免资金逻辑混乱
3. 支持不同币种对应不同充值 / 提现方式
4. 提供完整、可追溯的资金记录
5. 确保提现合规（KYC + 风控）

### 3. 适用范围

- **客户端**: iOS / Android / H5
- **用户角色**:
  - 普通用户
  - 主播（含签约 / 非签约）

### 4. 钱包总体设计

#### 4.1 钱包币种结构

- 钱包按**币种维度隔离**，每个币种一套独立余额与账本
- 当前支持币种：**BSB**（锚定汇率: 1BSB = 0.1USDT）、**VND**、**IDR**、**THB**
- 金额展示规则：

| 币种 | 千分位规则 | 小数点规则 |
|------|----------|----------|
| VND  | `.`（点）  | 整数     |
| IDR  | `.`（点）  | 整数     |
| THB  | `,`（逗号）| 2位小数   |
| BSB  | `,`（逗号）| 2位小数   |

#### 4.2 币种切换规则

- 钱包首页可切换当前币种
- **当前选中币种 = 所有操作的唯一数据源**：充值、提现、兑换、交易记录
- 切换币种即切换：余额、充值/支付方式、记录列表数据
- **不支持跨币种混合展示与操作**

#### 4.3 页面交互

- 点击切换：跳转选择币种界面
- 点击充值：温馨提示弹窗
- 继续充值：充值界面（仅支持USDT充值，不支持法币充值）
- 去登录：跳转登录界面（登录后返回原界面，再点击充值，展示所有充值方式）
- 点击记录：跳转该币种记录界面

### 5. 充值模块

#### 5.1 充值方式总览

| 类型     | 方式                      | 是否平台承载页 | 状态回调           | 备注                    |
|----------|--------------------------|-------------|-------------------|------------------------|
| 虚拟币   | USDT（TRC20/ERC20）       | 是          | 第三方支付通道回调   | 链上转账                |
| 银行     | 银行转账                   | 否          | 第三方支付通道回调   | 第三方通道              |
| 电子钱包 | QRIS / PromptPay 扫码      | 否          | 第三方支付通道回调   | 第三方通道二维码 + 倒计时 |

**充值通用规则**：
1. 每个币种进来有默认充值方式及默认最小档位快捷金额
2. 充值金额输入框支持手动填写（手动填写数额不得小于最低档位）
3. 点击去充值后生成订单匹配通道
4. 只有BSB充值金额输入框下有对应汇率展示及兑换所得的金额

#### 5.2 USDT充值

**入口规则**：
- 仅在支持USDT支付方式的**币种**下可见
- 游客登录状态也支持USDT充值

**金额规则**：
- 充值单位：钱包币种单位
- 系统根据汇率自动换算USDT

**USDT支付页**：
- 网络选择：TRC20（默认），可扩展 ERC20 / BEP20
- 展示信息：网络类型、收款地址（可复制）、二维码
- 风险提示：
  - 复制收款地址或扫描二维码
  - 仅支持当前网络 USDT
  - 到账时间取决于区块链网络确认，通常 2-10 分钟

**到账机制**：
- 支付状态回传平台
- 通过**区块链监听服务**确认到账
- 达到确认数后自动上账

#### 5.3 银行转账充值

**入口规则**：根据当前币种展示对应国家银行转账

**流程**：
1. 用户选择充值金额
2. 点击"立即充值"
3. 平台向支付通道创建订单
4. 获取并展示转账信息（三方承载页）：银行名称、账户名、账号、金额、转账备注（UserID + 时间戳）
5. 支付状态回传平台
6. 平台更新订单状态

**回调说明**：
- "查看详情"用于：查看订单状态
- "继续转账程序"用于：返回三方台面

#### 5.4 QRIS / PromptPay 扫码充值

**入口规则**：印尼 / 泰国法币钱包下展示，默认推荐方式

**承载页（三方）**：展示二维码（通道返回）、倒计时、支付方式、金额（可复制）、备注（UserID + 时间戳）

**状态规则**：支付状态回传平台，平台更新订单状态

#### 5.5 重复转账拦截规则

**拦截触发逻辑**：
- 识别维度：**同一 UserID + 同一币种 + 同一充值金额**
- 状态定义：订单处于"待支付"或"进行中"
- 规则适用：银行转账、电子钱包支付（不适用于USDT链上充值）

**分级提示规则**：

| 待支付订单数 | 弹窗文案内容 | 按钮1（左） | 按钮2（次） | 交互 |
|------------|-----------|-----------|-----------|------|
| 1-2笔      | 您有N笔同等金额交易正在进行中，建议您先处理已有订单，是否确认继续创建新充值？ | 查看详情 | 继续充值 | 默认高亮"继续充值" |
| 3笔（上限） | 您有3笔同等金额的交易正在进行中，为避免重复扣款，建议更换其它方式进行充值 | 查看详情 | 确认 | 确认按钮高亮 |

**按钮规则与跳转**：
- **查看详情**：跳转至"充值记录"列表页
- **继续充值**：仅在 N < 3 时出现，点击后关闭弹窗并进入支付/下单流程
- **确认**：在 N = 3 时出现，点击后关闭弹窗，停留在充值中心页面，不触发订单创建

#### 5.6 额外奖金选择规则

**入口与展示**：
- 组件位置：位于"余额展示区"下方、"支付方式"上方
- 交互逻辑：默认状态显示"无奖金"或当前最优匹配活动
- 点击弹出"请选择您的奖金"半屏浮窗
- 浮窗内容：列出所有可用奖金活动，提供"无奖金"选项
- 底部强制提示："您的充值金额与奖金将入账充值钱包，需完成 xx 倍流水要求"

**奖金与充值金额联动**：
- 快捷金额位标签：选择特定奖金活动后，充值金额区域展示对应的快捷标签
- 标签右上角需带奖金比例角标（如 +50%）
- 选择或输入充值金额后，输入框右侧展示"额外 + XXX"字样
- 计算公式：**额外奖金 = 充值金额 × 奖金比例**

**门槛校验**：
- 若用户手动输入的金额低于所选奖金活动的最低充值门槛，点击"去充值"时弹气泡提示
- 状态互斥：同一笔充值订单仅限参与一个奖金活动

#### 5.7 充值页面交互

- 左上角：跳转返回个人中心
- 右上角：跳转客服中心
- 点击切换：币种切换（余额跟随币种变化，个人中心币种不受影响）
- 点击提现：弹出KYC认证温馨提示
- 点击活动奖金充值：弹出"请选择您的奖金"半屏浮窗
- 切换其他支付方式：底下的档位根据支付方式会发生变化
- 点击去充值：跳转三方承载页 / USDT 支付页
- 查看详情：跳转充值记录界面
- 继续转账程序：返回三方承载页

### 6. 兑换模块（法币 → BSB）

#### 6.1 兑换范围
- 支持：VND / IDR / THB → BSB（BSB充值界面无兑换icon）
- 仅限当前币种余额兑换

#### 6.2 汇率规则
- 统一展示口径：**1 BSB = X 法币**
- 示例：1 BSB = d2,620（越南盾）/ Rp 1,530（印尼盾）/ B3.50（泰铢）

#### 6.3 兑换计算
- 兑换金额：输入当前法币金额（超出余额提示：已达当前可兑换上限）
- 兑换获得 BSB xxx.xx
- 赠送奖励（如 10%）
- **实际到账 BSB = 兑换获得 + 赠送**

#### 6.4 赠送规则
- 赠送部分需完成 **X 倍流水**后可提现
- 在页面明确提示

#### 6.5 兑换页面交互
- 左上角：返回该币种充值中心界面
- 点击记录：跳转记录界面
- 点击兑换：3S内兑换成功提示

### 7. 提现模块

#### 7.1 提现总原则

**KYC与提现关系**：

| 选项       | 触发动作          | 最终权限                                      |
|-----------|-----------------|----------------------------------------------|
| "去认证"   | 进入KYC认证页面    | 审核通过后，解锁法币提现 / 电子钱包 / USDT 等全渠道 |
| "继续提现" | 跳过KYC认证       | 仅限使用 USDT 提现                              |

- **未认证状态**：点击提现触发弹窗，提供"去认证"和"继续提现"两个入口
- **已认证状态**：点击提现直接进入提现方式选择页，无弹窗拦截

**提现规则**：
- 法币充值 → 法币提现
- USDT充值 → USDT提现
- 法币 + USDT充值 → 法币提现
- 法币 → BSB → 法币提现

**其他规则**：
- 首次提现需输入账户信息
- 提现账户信息平台记录
- 后续提现无需重复填写（可更改提现信息）
- 账户持有人姓名 = KYC认证姓名（提现页自动填充已实名的姓名且不可编辑）
- 账户信息未填写完成 → 确认提现icon置灰
- 确认提现校验：输入金额超出每日提现限额/单笔最低时弹窗提示

#### 7.2 银行卡提现

**首次提现填写字段**：银行名称、银行账号、账户持有人（置灰，默认KYC名字）、开户网点、提现金额（手动输入，支持最大金额快捷输入）
**提示**：提现金额、手续费、到账金额

**非首次提现**：自动带出账户信息（只读），仅输入提现金额

#### 7.3 电子钱包提现

- 支持：GoPay / OVO / DANA / PromptPay 等
- 首次提现填写：钱包类型、手机号、账户持有人（置灰）、提现金额
- 后续提现复用（只读）

#### 7.4 BSB 提现

- 支持：USDT / 各大银行 / GoPay / OVO / DANA / PromptPay 等
- 只有BSB提现金额输入框下有对应**汇率展示**
- 后续提现复用（只读）

#### 7.5 提现限制（后台配置）

- 单笔限额
- 单日限额
- 手续费规则
- T+N 到账时间

#### 7.6 提现页面交互

- 左上角：返回该币种充值中心界面
- 点击记录：跳转记录界面
- 选择提现方式：跳转该方式提现信息输入页面
- 点击确认提现：弹出"提现状态"半屏浮窗
- 点击查看详情：跳转提现记录

#### 7.7 姓名一致性校验规则

- 账户持有人名称必须与 KYC 认证姓名一致
- **忽略大小写**
- **忽略多余空格**
- 不做完全等值字符串比对

### 8. 记录模块

#### 8.1 记录分类
- 提现 / 充值 / 兑换 / 奖励

#### 8.2 核心规则
- 钱包当前币种 = 记录当前币种
- 切换币种即切换整套记录数据
- **不支持跨币种混合记录**

#### 8.3 列表页面

- **提现字段**：金额、方式、时间、状态（颜色区分：已完成 / 审核中黄色 / 失败红色）、操作
- **充值字段**：金额、方式、时间、状态（颜色区分：已完成 / 审核中黄色 / 失败红色）、操作
- **兑换字段**：兑换金额、赠送x%、实际到账、时间
- **奖励字段**：奖励金额、奖励类别、时间

#### 8.4 详情页

- 所有金额币种**必须与列表页币种一致**
- 展示字段：订单号、金额明细、状态、时间、方式、失败原因（如有）
- 充值失败原因：支付超时、支付异常、卡片问题
- 提现失败原因：信息有误、系统维护、额度受限

### 9. 非功能性需求

- **对账一致性**：明确"账务流水"与"账户余额"的强一致性校验机制
- **幂等性**：充值回调必须做幂等处理，防止重复入账

### 10. 风控与异常处理

- KYC 强制校验
- 提现账户白名单
- 姓名强一致校验
- 不支持用户自行修改提现账户
- 异常情况需人工介入
- 大额 / 异常频率触发人工审核

## 二、产品原型分析

### 1. 三方充值页面（QRIS支付）

- 页面标题："QRIS 支付"
- 显示倒计时（如 04:56），倒计时结束显示红色"订单失效"
- 展示二维码，提示"使用任意电子钱包或银行 App 扫码"
- 字段：支付方式QRIS、金额RP 3,000,000（带COPY按钮）、备注UserID+时间戳（带COPY按钮）
- 底部提示：打开支持QRIS的电子钱包 → 扫描二维码 → 自动到账

### 2. 个人中心

**已登录状态**：
- 显示用户昵称、ID（如 44038777）、等级标识
- 钱包余额区：显示币种切换icon、当前余额（如 BSB 1,000,000.00）
- 操作按钮："记录"、"充值"
- "我的权益"：贵族、活动、推广
- 首充礼包横幅广告
- 菜单列表：我的等级、我的背包、浏览历史、游戏历史、主播中心

**币种切换页面**：
- 列出所有币种及对应余额（带国旗icon标识和选中勾号）

**游客状态点击充值**：
- 弹出"温馨提示"弹窗："当前为游客状态，登录后解锁更多充值方式"
- 按钮："继续充值"、"去登录"

### 3. 充值各币种页面

**BSB充值页面**：
- 余额显示：BSB100.00，操作按钮仅"提现"（无兑换）
- 奖金选择：无奖金（下拉）
- 支付方式：USDT（标"推荐"）TRC20/ERC20、越南银行、电子钱包扫码
- 快捷金额档位：100, 200, 500, 1,000, 2,000, 5,000, 10,000, 50,000, 100,000
- 底部汇率：1BSB=0.1USDT

**越南（VND）充值页面**：
- 操作按钮："兑换"、"提现"
- 支付方式：越南转账
- 快捷金额：200,000 / 500,000 / 1,000,000 / 3,000,000 / 5,000,000 / 10,000,000

**印尼（IDR）充值页面**：
- 操作按钮："兑换"、"提现"
- 支付方式：电子钱包扫码（标"推荐"）、印尼银行
- 快捷金额：150,000 / 300,000 / ... / 150,000,000

**泰国（THB）充值页面**：
- 操作按钮："兑换"、"提现"
- 支付方式：电子钱包扫码（标"推荐"）、泰国银行
- 快捷金额：300.00 / 700.00 / ... / 350,000.00

### 4. USDT支付页

- 页面标题："USDT 支付"（平台承载页）
- 网络选择：TRC20（下拉可选 ERC20 / BEP20）
- 收款地址展示（带COPY按钮）+ 二维码
- 底部提示3点

### 5. 切换充值方式

- BSB选择USDT充值：汇率 1BSB=0.1USDT
- BSB选择越南银行：汇率 1BSB=d2,620
- BSB选择电子钱包：汇率 1BSB=Rp1,530

### 6. 选择奖金

- 奖金选择浮窗：列出可用奖金活动 + "无奖金"选项
- 红色提示："需完成*xx倍流水要求"
- 选择后：快捷金额标签带角标（+50%），输入框右侧显示"额外+500"

### 7. 通用回调

- 银行转账回调页：充值进行中弹窗（充值方式、订单号、金额、时间）
- 重复充值1-2笔：提醒弹窗（查看详情 / 继续充值）
- 重复充值3笔：拦截弹窗（查看详情 / 确认）

# 第二部分：币种配置（主责模块）

## 三、需求文档分析

### 1. 文档背景与目标

**背景**：钱包体系作为平台的核心基础能力之一，用于承载用户在平台内的资金流转与消费行为，需同时满足娱乐消费与 iGaming 投注两类业务场景。

**目标**：本体系通过清晰的钱包类型划分与严格的资金流向规则，确保用户体验一致，资金安全可控，并支持后续业务财务、风控能力的扩展。

**文档规范**：
- **绿字**：解释说明内容
- **红字**：需要特别注意的内容
- **灰字**：需要了解，但当前版本暂不制作的内容
- **删除线**：删除、移除的内容

### 2. 资金钱包体系

#### 2.1 资金钱包定义
- 资金钱包用于承载用户在平台内的**真实资金**，用于 iGaming 投注、结算、兑换钻石及提现等操作
- 资金钱包支持**平台币、多法币、多虚拟币**，C端钱包中各币种金额**相互独立**
- 平台币暂命名为：**BSB**

#### 2.2 使用场景与获得途径

**使用场景**：
- 娱乐场消费：刷礼物、购买虚拟道具、购买特权及付费解锁内容等（娱乐场消费也可稽核）
- 投注：电子、真人视讯、彩票、体育中投注
- 提现：法币充值可提法币，虚拟币充值可提虚拟币；法币、虚拟币混合充值只可提法币

**获得途径**：
- 充值：直充法币或虚拟币
- 投注返奖：通过投注场景赚取的资金
- 活动/任务返奖：参与游戏活动/任务获得的奖金

#### 2.3 资金钱包结构

**资金钱包余额 = 中心钱包余额 + 奖励钱包余额**

| 钱包类型 | 定义 | 资金来源 | 资金去向 |
|---------|------|---------|---------|
| 中心钱包 | 用户主资金钱包 | 充值/投注返奖/奖励完成稽核 | 投注（需完成稽核流水方可提现）→ 中心钱包（可提现） |
| 奖励钱包 | 活动/任务奖金暂存钱包 | 领取各活动（签到、任务、VIP、分销奖励） | 稽核完成 → 中心钱包 |
| 主播钱包 | 主播收益资金账户 | 礼物打赏、主播活动/任务奖励 | 可转入中心钱包（默认提现）、可提现 |
| 代理钱包（预留） | 代理分佣收益 | 代理分成 | 可转入中心钱包（默认提现）、可提现，转入后需稽核 |
| 场馆钱包（预留） | iGaming场馆临时资金 | 中心钱包+活动钱包转入 | 投注 → 退出场馆 → 中心钱包+活动钱包 |

#### 2.4 资金消费规则
- 消费时**优先扣除中心钱包余额，再扣奖励钱包余额**
- 当前游戏均为自研游戏，进入游戏后钱包余额 = 资金钱包余额
- 中心钱包余额投注返奖计入中心钱包，活动钱包余额投注返奖计入活动钱包
- **单次下注使用到两个钱包余额混合投注时，需根据投注金额占比分开计返奖**

### 3. 币种配置

#### 3.1 币种类型划分

| 币种类型 | 说明 | 示例 |
|---------|------|------|
| 法定货币 | 国家/地区官方货币 | IDR、VND、THB |
| 加密货币 | 区块链或平台支持的数字资产 | USDT、BTC、ETH |
| 平台货币 | 平台专属的货币 | BSB |

- **每个币种对应一个币种资金钱包，不同币种钱包互相独立**

### 4. 汇率规则

#### 4.1 汇率应用场景

| 场景 | 使用汇率 | 说明 |
|------|---------|------|
| 充值 | 入款汇率 | 跨币种充值时 |
| 提现 | 出款汇率 | 跨币种提现时 |
| 投注 | 入款汇率 | 进入游戏时转换，记录汇率，退出时原汇率转回 |
| 统计 | 平台汇率 | 报表统计换算为基准货币 |

#### 4.2 汇率计算公式

**实时汇率获取**：
- 通过**3家三方API**获取，算平均值作为实时汇率
- 频率可约定（测试期间约1天1次，上线按运营需求）

**平台汇率**：
- 平台汇率 = 平台对该币种刷新的实时汇率
- 初始：平台汇率为空时，直接取实时汇率值

**入款汇率与出款汇率**：
- **入款汇率 = 实时汇率 × (1 + 汇率浮动%)**
- **出款汇率 = 实时汇率 × (1 - 汇率浮动%)**

**偏差值与阈值机制**：
- **偏差值 = (实时汇率 - 平台汇率) / 实时汇率 × 100%**
- 当偏差值 >= 阈值% 时，更新平台汇率
- 偏差值判断需**同时判断入款和出款偏差值**，任一达到阈值即更新
- 未超过阈值的币种不更新

**更新举例**：
- BSB:VND = 1:71，平台70 → 实时71，偏差 = (71-70)/71 = 1.4%，若阈值2%，不更新
- 每次平台汇率更新后需**生成汇率日志**

#### 4.3 汇率展示

| 货币类型 | C端展示精度 | 系统侧精度 |
|---------|-----------|-----------|
| 法定货币 | 小数点后**2位** | 小数点后**2位**，不足补0，超过**四舍五入** |
| 加密货币 | 小数点后**8位** | 小数点后**8位**，不足补0，超过**四舍五入** |

#### 4.4 汇率日志

- 位于：操作日志页面增加**【汇率日志】**页签
- 当有币种的平台汇率变更时，生成一条日志

**汇率日志字段**：

| 字段 | 说明 |
|------|------|
| 序号 | 表格序号 |
| 日志编号 | 唯一编码 |
| 更新时间 | 日志生成时间 |
| 币种代码 | 更新汇率的币种 |
| 实时汇率 | 更新时的三方实时汇率 |
| 平台汇率 | = 更新时的三方实时汇率 |
| 入款汇率 | 根据平台汇率计算 |
| 出款汇率 | 根据平台汇率计算 |

**筛选条件**：更新时间（日期范围）、币种代码

**页面位置**：运营后台 → 系统管理 → 操作日志 → 汇率日志页签

# 第四部分：跨模块汇总

## 核心业务规则汇总

### 1. 币种隔离原则
- 所有操作（充值、提现、兑换、记录）严格按币种隔离
- 不支持跨币种混合展示与操作
- 每个币种对应独立的资金钱包

### 2. BSB锚定汇率
- **1 BSB = 0.1 USDT**（即 10 BSB = 1 USDT）
- 法币汇率：1BSB = d2,620（VND）/ Rp1,530（IDR）/ B3.50（THB）

### 3. 钱包优先级
- 消费优先：**先扣中心钱包，再扣奖励钱包**
- 返奖按比例：混合投注时按各钱包投注金额**占比分开计算返奖**

### 4. KYC分级权限
- 未认证：仅可USDT提现
- 已认证：解锁全渠道（法币/电子钱包/USDT）
- 提现账户持有人 = KYC认证姓名（忽略大小写和多余空格）

### 5. 重复充值拦截
- 维度：同用户 + 同币种 + 同金额
- 1-2笔：提醒可继续
- 3笔：强制拦截

### 6. 奖金流水要求
- 充值奖金和兑换赠送部分需完成 **X 倍流水**后方可提现

### 7. 幂等性保障
- 充值回调必须幂等处理，防止重复入账

### 8. 提现审核流程（多级）
- 风控审核 → 财务审核 → 出款审核
- 支持锁定/解锁/强制解锁操作

## 关键公式汇总

| 公式 | 说明 |
|------|------|
| 资金钱包余额 = 中心钱包余额 + 奖励钱包余额 | 钱包余额计算 |
| 10 BSB = 1 USDT | BSB锚定 |
| 入款汇率 = 实时汇率 × (1 + 汇率浮动%) | 充值汇率 |
| 出款汇率 = 实时汇率 × (1 - 汇率浮动%) | 提现汇率 |
| 偏差值 = (实时汇率 - 平台汇率) / 实时汇率 × 100% | 汇率偏差 |
| 当偏差值 >= 阈值%时，触发平台汇率更新 | 阈值触发 |
| 额外奖金 = 充值金额 × 奖金比例 | 充值奖金 |
| 成功率 = 成功订单数 / 总订单数 × 100% | 通道统计 |
| 手续费 = 费率 × 成功订单金额 | 手续费计算 |
| 净入账金额 = 成功订单金额 - 手续费 | 净入账 |
