# 多币种钱包需求分析总结

## 一、币种配置模块（基础设施层）

### 1.1 资金钱包体系
- 用户拥有**4类钱包**：中心钱包、奖励钱包、代理钱包、主播钱包
- 每个钱包下按币种细分子钱包，每个子钱包独立记账
- 钱包间关系：充值进入中心钱包，奖励进入奖励钱包，各钱包余额独立，但展示时合并为该币种总余额

### 1.2 币种类型划分
- **法币**：VND（越南盾）、IDR（印尼盾）、THB（泰铢）等
- **虚拟币**：USDT（TRC20/ERC20/BEP20）
- **平台币**：BSB（平台基准币种）
- BSB作为平台基准币种，所有法币/虚拟币都有对BSB的汇率

### 1.3 后台配置页面
- 币种列表管理：币种名称、符号、图标、小数位、状态（启用/禁用）
- 支持配置充值档位、充值范围、提现限额、手续费等
- 操作日志记录所有配置变更

### 1.4 汇率体系
- **应用场景**：充值换算、兑换BSB、提现换算、后台统计展示
- **汇率规则**：支持手动设置平台汇率，汇率变更有日志记录
- **汇率展示**：前端展示当前币种对BSB的实时汇率
- **汇率日志**：记录每次汇率变更的时间、操作人、变更前后值

## 二、多币种钱包模块（核心业务层）

### 2.1 钱包总体设计
- 用户个人中心展示当前选中币种的钱包余额
- 支持币种切换，切换后整套数据（余额、记录）随之切换
- 右上角币种切换入口，列出所有可用币种及各币种余额
- 游客状态下可充值BSB，但提示登录解锁更多功能

### 2.2 币种切换规则
- 切换币种 = 切换整套钱包视图（余额、充值方式、记录等）
- 不支持跨币种混合记录

### 2.3 充值模块
**充值方式总览（按币种不同）**：
- **BSB**：USDT（推荐）、银行转账、电子钱包扫码，显示汇率换算
- **VND**：越南转账
- **IDR**：电子钱包扫码（推荐）、印尼银行
- **THB**：电子钱包扫码（推荐）、泰国银行

**USDT充值**：选择网络（TRC20/ERC20/BEP20）→ 显示收款地址+二维码 → 链上确认到账

**银行充值**：跳转三方支付页面 → 回调更新状态

**扫码充值**：QRIS等 → 倒计时5分钟 → 超时订单失效

**重复转账拦截**：
- 1笔同等金额进行中 → 提示是否继续
- 3笔同等金额进行中 → 建议更换方式

**额外奖金选择**：充值时可选活动奖金（如额外50%），选择后档位金额更新，充值金额有最低要求，奖金进入奖励钱包需完成流水要求

### 2.4 兑换模块
- 法币/虚拟币 → BSB 的单向兑换
- 输入兑换金额 → 按汇率计算获得BSB → 赠送x%（进奖励钱包）→ 显示实际到账
- 赠送部分需完成倍流水后可提现

### 2.5 提现模块
**提现规则（核心）**：
- 法币充值法币 → 法币提现
- USDT充值 → USDT提现
- 法币+USDT充值 → 法币提现
- 法币→BSB → 法币提现

**提现方式**：
- **银行卡提现**：T+1工作日，填写银行名称、账号、持有人（KYC姓名置灰）、开户网点
- **电子钱包提现**：即时到账，GoPay/OVO/DANA/ShopeePay/PromptPay等
- **BSB提现**：支持多种方式，金额输入框下有对应汇率展示
- **USDT提现**：即时到账

**提现前置条件**：
- 首次提现需KYC认证（姓名、国家、手机号、证件、上传证件照）
- 未KYC → 弹窗提示去认证，仅可USDT提现
- 账户持有人姓名 = KYC认证姓名（不可编辑）
- 姓名一致性校验（忽略大小写、多余空格，不做完全等值字符串比对）

**提现限制（后台配置）**：单笔限额、单日限额、手续费规则、T+N到账时间

**提现校验**：超出每日限额/低于单笔最低 → 弹窗提示

### 2.6 记录模块
- **分类**：提现、充值、兑换、奖励 四个Tab
- **核心规则**：钱包当前币种 = 记录当前币种，切换币种即切换整套记录
- **列表页**：
  - 提现：金额、方式、时间、状态（已完成/审核中/失败）、操作
  - 充值：金额、方式、时间、状态、操作
  - 兑换：兑换金额、赠送x%、实际到账、时间
  - 奖励：奖励金额、奖励类别、时间
- **详情页**：订单号、金额明细、状态、时间、方式、失败原因

### 2.7 非功能性需求
- 对账一致性：账务流水与账户余额强一致性校验
- 幂等性：充值回调必须幂等处理，防止重复入账

### 2.8 风控与异常处理
- KYC强制校验、提现账户白名单、姓名强一致校验
- 不支持用户自行修改提现账户
- 异常情况需人工介入
- 大额/异常频率触发人工审核

## 三、财务管理模块（他人负责，关注依赖点）

### 3.1 通道配置
- 代收通道（充值用）、代付通道（提现用）
- 每个通道：币种、通道名称、支付类型、费率、单笔限额、权重、预警金额、状态
- 通道轮询规则：成功率系数、时间系数、预警系数

### 3.2 支付配置
- 充值方式配置：币种、充值方式、支付图标、充值档位/范围、关联通道、排序、状态
- 提现方式配置：类似结构

### 3.3 充值记录（后台）
- 完整充值流程图：客户端→服务端→通道 三方交互
- 字段涵盖：订单号、用户、通道、币种、充值方式、状态、金额、活动、汇率等
- 支持基准币种换算展示

### 3.4 提现记录（后台）
- 完整提现流程图：含风控审核→财务审核→出款审核 三级审核
- 支持锁定/解锁/强制解锁操作

### 3.5 人工修正
- **充值补单**：关联原订单、补单金额、赠送金额、需审核
- **人工加款**：选币种→对4类钱包分别设置加款金额，需审核
- **人工减款**：类似加款，支持异常追回/风控处罚/财务冲正/费用调整

### 3.6 数据统计
- 通道统计、充值统计、提现统计
- 统计数据实时更新，支持时区配置（UTC基准）

## 四、模块间关键依赖关系

| 依赖方向 | 说明 |
|---------|------|
| 钱包 → 币种配置 | 钱包需要读取币种列表、汇率、充值档位、提现限额等配置 |
| 钱包 → 财务(支付配置) | 充值时需获取该币种可用的充值方式及关联通道 |
| 钱包 → 财务(通道) | 充值下单需通过通道轮询匹配可用通道 |
| 财务(充值记录) → 钱包 | 充值回调成功后需调用钱包服务进行入账 |
| 财务(提现记录) → 钱包 | 提现审核通过+出款成功后需扣减钱包余额（或提现申请时先冻结） |
| 财务(人工修正) → 钱包 | 人工加款/减款/补单审核通过后需调用钱包服务修改余额 |
