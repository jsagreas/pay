# ser-wallet 需求分析汇总

---

## 一、你的核心职责：多币种钱包模块

### 1.1 钱包整体架构

| 维度 | 说明 |
|------|------|
| **钱包类型** | 中心钱包（主）、奖励钱包（需稽核/流水达标才能转入中心）、主播钱包、代理钱包（预留）、场馆钱包（预留） |
| **资金钱包余额** | = 中心钱包 + 奖励钱包 |
| **消费优先级** | 中心钱包优先，不足部分扣奖励钱包 |
| **币种隔离** | 每个币种独立钱包，切换币种 = 切换所有操作的数据源 |

### 1.2 支持的币种体系

| 类型 | 币种 | 说明 |
|------|------|------|
| 平台币 | BSB | 10 BSB = 1 USDT（锚定） |
| 法币 | VND、IDR、THB | 面向不同地区用户 |
| 加密货币 | USDT | TRC20/ERC20/BEP20 三条网络 |

### 1.3 四大功能模块

**充值模块：**
- USDT充值（三条网络选择）、银行转账、QR扫码（QRIS/PromptPay）
- 重复转账拦截：同UserID + 同币种 + 同金额，1-2笔待处理=警告，3笔=阻断
- 奖金活动选择：关联活动、最低充值门槛校验、稽核倍数要求

**兑换模块：**
- 方向：法币（VND/IDR/THB）→ BSB，单向
- 汇率展示 + 10%兑换奖金（需X倍流水）

**提现模块：**
- 方式：银行卡、电子钱包（GoPay/OVO/DANA/ShopeePay/PromptPay）、USDT
- KYC：USDT提现可选，法币/电子钱包提现必须
- 提现规则：法币充→法币提；USDT充→USDT提；混合充→仅法币提
- 限额：单笔/每日限额、手续费、T+N到账周期

**记录模块：**
- 充值/提现/兑换/奖励四类记录，含状态追踪

### 1.4 非功能要求
- 幂等性、一致性、风控规则

## 二、你的核心职责：币种配置模块（B端后台）

### 2.1 钱包系统结构
- 5种钱包类型的定义和管理规则
- 基准币种：USDT（配置一次后不可变）

### 2.2 汇率体系

| 汇率类型 | 计算方式 |
|----------|----------|
| 实时汇率 | 3个第三方API取平均值 |
| 平台汇率 | 基于实时汇率，阈值触发更新：\|实时-平台\|/实时*100% > 阈值 → 更新 |
| 充值汇率 | 实时汇率 × (1 + 浮动%) |
| 提现汇率 | 实时汇率 × (1 - 浮动%) |

### 2.3 后台配置页
- 币种列表（增删改查、icon上传）
- 汇率管理（浮动比例配置）
- 汇率变更日志

## 三、依赖参考：财务管理模块（他人负责）

### 3.1 模块结构

```
财务管理
├── 通道配置（代收通道 / 代付通道 / 轮询规则）
├── 支付配置（充值方式 / 提现方式）
├── 充值记录（订单全流程 + 导出密码校验）
├── 提现记录（三级审核：风控→财务→出款 + 锁定/解锁）
├── 人工修正（补单B+ / 加款A+ / 减款M+，双人复核）
└── 数据统计（通道/充值/提现统计，UTC时区转换）
```

### 3.2 关键业务规则
- **订单号规则**：充值C+16位、提现T+16位、补单B+16位、加款A+16位、减款M+16位
- **通道轮询**：按权重分配，3个系数（成功率/时间/预警），不同币种独立规则
- **提现三级审核**：风控审核 → 财务审核 → 出款审核
- **人工修正**：创建方 ≠ 审核方，分钱包维度操作（中心/奖励/代理/主播）
- **导出安全**：密码 + 可选谷歌验证码

## 四、模块间依赖关系分析

这是最关键的部分——三个模块之间的接口依赖：

| 调用方 | 被调用方 | 依赖内容 |
|--------|----------|----------|
| **多币种钱包（C端）** → **币种配置（B端）** | 获取币种列表、汇率（充值/提现/平台汇率）、BSB锚定价 |
| **多币种钱包（C端）** → **财务管理** | 充值时：通过支付配置获取可用充值方式→关联代收通道→轮询分配；提现时：创建提现订单→进入三级审核流程 |
| **财务管理** → **多币种钱包** | 人工加款/减款需要操作用户的各钱包余额；补单需要给用户钱包上分 |
| **财务管理** → **币种配置** | 通道配置按币种管理；统计数据需要平台汇率做基准币种换算 |

**核心依赖链路：**
1. **充值链路**：用户选币种 → 钱包获取充值方式（来自支付配置）→ 提交订单 → 轮询匹配通道 → 创建订单（财务管理）→ 通道回调 → 钱包上分
2. **提现链路**：用户选币种 → 钱包获取提现方式 → KYC校验 → 提交订单 → 三级审核（财务管理）→ 通道出款 → 钱包扣款
3. **兑换链路**：用户选法币 → 获取平台汇率（币种配置）→ 计算BSB金额 → 扣法币钱包 → 加BSB钱包

## 五、财务管理需求文档详细解析

### 5.1 文档说明

**功能列表：**
1. 通道配置
2. 支付配置
3. 充值记录
4. 提现记录
5. 人工修正
6. 通道统计
7. 充值统计
8. 提现统计

**文档背景：** 随着平台支付通道及资金体系的复杂度提升，需要通过统一的财务管理与数据统计能力，对支付配置、资金流水及相关数据进行规范化管理与可视化呈现，为资金安全、对账审计及业务分析提供支撑。

**产品目标：** 构建统一、可审计的财务管理与数据统计模块，覆盖支付通道配置、充值提现记录管理及关键资金数据统计分析，提升平台资金管理效率、数据准确性与决策支持能力。

**使用对象：** 公司内部研发团队（产品经理、交互/UI设计师、后端/前端开发工程师、测试工程师）

**文档规范：**
- 绿字：解释说明内容
- 红字：需要特别注意的内容
- 灰字：需要了解，但当前版本暂不制作内容
- 删除线：删除、移除的内容

### 5.2 通道配置

#### 5.2.1 代收通道

**搜索字段：** 通道（下拉全部）、币种（大写英文简称）、通道状态（启用/禁用）

**列表字段：**
| 字段 | 说明 |
|------|------|
| 通道名称 | 创建时设定，不可修改 |
| 余额 | 通道代币余额 |
| 通道费率 | 千分比表示 |
| 最小/最大金额 | 用户代收交易金额限制 |
| 代付通道 | 关联的代付通道 |
| 通道上限 | 日交易上限金额 |
| 代收总笔数 | 统计信息 |
| 创建时间 | 创建通道时间 |
| 状态 | 启用/禁用 |

**编辑弹窗字段：** 通道名称、通道标识、币种、费率、最小/最大金额、代付通道（下拉选择）、通道上限、状态

#### 5.2.2 代付通道

与代收通道类似，额外字段：
- 中间钱包余额：标记地址中余额情况
- 半自动出款：标记代付通道是否需要半自动出款

#### 5.2.3 通道轮询规则

- 轮询配置限定于代收通道
- 本期支持方式：**按权重（Weight）轮询**
- 所有轮询规则在同一币种内生效，**不同币种独立轮询规则**
- 权重轮询：总权重由各通道配置权重之和确定，无需保证合等于100，只关注比例分配
  - 例如：通道A权重7、通道B权重3，则约70%和30%
- 仅**启用状态**的通道参与轮询计算
- 轮询规则被禁用 = 无通道可选用，订单流走默认策略

#### 5.2.4 操作日志
记录代收/代付通道的编辑、启用、禁用、轮询规则操作

### 5.3 支付配置

#### 5.3.1 充值方式

**列表字段：** 充值方式名称、代收通道、充值方式类型、币种、大额通道、充值金额（最小/最大）、头像icon、排序、状态

**编辑弹窗字段：** 充值方式名称、代收通道（下拉选择）、充值方式类型、币种、大额通道、最小/最大充值金额、头像上传（PNG，≤2.5M）、排序（数值越小越靠前）、状态

#### 5.3.2 提现方式

**列表字段：** 提现方式名称、代付通道、提现方式类型、币种、头像icon、排序、状态

**编辑弹窗字段：** 提现方式名称、代付通道、提现方式类型、币种、头像上传、排序、状态

### 5.4 充值记录

**列表字段（18列）：**
序号、订单号、三方订单号、充值方式、平台订单号、用户UID、用户编号、代收通道、币种、充值金额、实际到账、手续费、订单来源（Android/iOS/H5）、订单状态、创建时间、充值时间、充值回调时间、备注

**充值流程（三泳道：客户端→服务端→通道）：**
1. 用户选择充值目标币种
2. 选择/切换充值方式
3. 选择/输入充值金额
4. 服务端轮询匹配对应代收通道
   - 未命中 → 提示通道繁忙/金额限制，请更换充值方式
   - 命中 → 创建订单，状态「待支付」
5. 通道创建通道充值订单号
6. 客户端拉起支付页面
7. 通道回传支付结果
   - 失败 → 订单标记「支付失败」
   - 成功 → 订单「已完成」并记入用户上分

**导出功能优化：** 后台导出统一增加密码校验（账号密码 + 可选谷歌验证码）

### 5.5 提现记录

**列表字段（24列）：**
序号、订单号（T+16位）、三方订单号、提现方式、平台订单号、用户UID、用户编号、代付通道、币种、提现金额、实际到账、通道手续费、平台手续费、订单来源、中间钱包地址、收款账号信息、订单状态、锁定状态、风控审核时间、财务审核时间、出款审核时间、提现创建时间、提现成功时间、备注

**订单状态枚举：** 风控审核中、财务审核中、出款审核中、出款中、已完成、已取消、已拒绝、已锁定

**提现流程（三泳道：客户端→服务端→通道）：**
1. 用户选择提现币种
2. 实名认证判断（未认证→引导认证）
3. KYC提现条件判断
4. 选择/切换提现方式
5. 选择/输入提现金额
6. 提交订单
7. **三级审核：**
   - 风控审核 → 不通过=拒绝
   - 财务审核 → 不通过=拒绝
   - 出款审核 → 不通过=人工介入
8. 通过后向通道发起出款请求
9. 通道回传出款结果
10. 成功→订单「已完成」并扣除用户余额

### 5.6 人工修正

#### 5.6.1 充值补单
- 订单号：B+16位随机数
- 输入充值订单号或三方订单号查询，系统自动回填充值信息
- 一个订单号只能关联一个补单，不可重复
- **审核规则：创建方 ≠ 审核方**，需运营总监以上确认
- 审核通过后金额加入用户余额

#### 5.6.2 人工加款
- 订单号：A+16位随机数
- 选择加款类型（赠送奖金、活动补偿等）
- 指定用户UID/编号，输入加款金额（仅正数）
- 必须输入备注说明原因
- **审核规则：创建方 ≠ 审核方**

#### 5.6.3 人工减款
- 订单号：M+16位随机数
- 选择减款类型
- 余额不足时提示无法减款
- **审核规则：创建方 ≠ 审核方**

### 5.7 数据统计

**通用规则：**
- 数据实时更新，服务器以UTC基准时间记录
- 管理后台可设定站点时区，查询/配置时间以站点时区为准
- 时区转换：站点时区 → 还原UTC+0 → 查询数据库
- 时区设置支持IANA Time Zone标准

#### 5.7.1 代收通道统计
- 汇总卡片：代收总金额、通道手续费总额、代收成功总额、代收订单总数
- 统计字段：通道名称、时间维度、币种、代收金额、订单数、成功订单、成功金额、成功率、通道手续费、利润

#### 5.7.2 代付通道统计
- 汇总卡片：代付总金额、通道手续费、代付成功总额、代付订单数
- 统计字段：与代收类似（代付金额、代付订单数、代付成功率等）

#### 5.7.3 充值统计
- 汇总卡片：充值总金额、充值成功总额、充值订单总数
- 统计字段：充值方式、币种、充值金额、充值入账、订单数、成功订单、成功金额、人均充值金额、充值成功率

#### 5.7.4 提现统计
- 统计字段：提现方式、币种、提现金额、提现入账、订单数、成功订单、成功金额、收款账号、中间钱包地址、平台手续费、通道手续费、人均提现金额、提现成功率
