### 四、完整请求链路（HTTP → RPC → DB → 响应）

```
客户端 HTTP 请求
    ↓
Hertz网关（gate-back 或 gate-font）
    ↓ 全局中间件：HertzAccLogStr()（打印trace_id日志）
    ↓
路由组中间件（按顺序执行）：
    ├─ SetHeaderInfo()    → 提取IP/Token/Lang/DeviceID/DeviceType到metainfo
    ├─ WhiteCheck()       → [B端] 调用rpc.IpClient().OnWhiteList()
    │  BlackCheck()       → [C端] 调用rpc.IpClient().OnBlackList()
    └─ BackAuthMiddleware → [B端/admin/api] Token→Redis→BUserTokenPayload + CheckUrlAuth(path)
       FontAuthMiddleware → [C端/api] Token→Redis(RedisTokenKey+token)→CUserTokenPayload
    ↓
Handler函数（99%为一行代理）：
    rpc.Handler[REQ,RESP](ctx, c, rpc.XxxClient().Method)
    ↓ 内部流程：
    ├─ 反射初始化REQ对象（处理指针/非指针类型）
    ├─ c.BindAndValidate(&req)  → 请求参数绑定+校验
    ├─ 执行RPC调用              → Kitex Client → ETCD服务发现 → 目标服务
    └─ ret.HttpOne()            → 统一响应封装
       ├─ 成功：{traceId, code:0, msg:"", data: resp}
       └─ 失败：识别kerrors.BizStatusError → i18n翻译msg → {traceId, code:xxx, msg:"翻译后消息"}
    ↓
RPC服务端（ser-xxx）
    ↓ 服务端中间件：打印请求日志（TraceID/包名/服务名/方法名/耗时/来源）
    ↓
handler.go → 纯分发到Service
    ↓
ser层（业务逻辑）：参数校验 → 业务规则 → 调用rep层 → 组装响应
    ↓
rep层（数据访问）：调用gorm_gen/query类型安全API → TiDB
    ↓
返回响应（原路返回）
```

**Handler三个泛型变体**（`common/rpc/handler.go`）：
1. `Handler[REQ,RESP]` — 标准：绑定参数+调用RPC+返回JSON
2. `HandlerNoParam[RESP]` — 无参数：跳过绑定直接调用
3. `HandlerWithCSV[REQ,RESP]` — CSV导出：检查CSVResponse接口，设置Content-Disposition头

### 五、鉴权体系详解

#### B端鉴权（BackAuthMiddleware）
```
1. 从context获取Token（header优先，cookie次之）
2. Token为空 → 返回TokenNotExist
3. Redis查询：Key = token字符串（无前缀）
4. 反序列化为 session.BUserTokenPayload{ID, UserName}
5. ID为0 → 返回TokenAnalysisFailed
6. 调用 rpc.BUserClient().CheckUrlAuth(ctx, c.FullPath())
   → 超管/系统管理员(super_admin_flag=1/2)直接通过
   → 普通角色查询sys_role_menu+sys_menu表匹配path
7. 权限不足 → 返回CheckUrlAuthFailed
```

#### C端鉴权（FontAuthMiddleware）
```
1. 从context获取Token
2. Token为空 → 返回TokenNotExist
3. Redis查询：Key = RedisTokenKey + token（有前缀"user:token:"）
4. 反序列化为 session.CUserTokenPayload{ID, UID, NickName, Device...}
5. ID为0 → 返回TokenAnalysisFailed
6. 无URL级权限检查（只验登录状态）
```

#### B端登录流程（ser-buser Login）
```
IP白名单校验 → 用户名查库 → 账号禁用检查 → 密码校验(salt+hash)
→ Google TOTP验证(可选,totp库) → Token生成(32位随机) → Redis存储(30天TTL)
→ 操作日志(ser-blog RPC) → 返回{Token, BindGoogle, RoleId, RoleName}
```

### 六、RBAC权限模型（ser-buser）

**四表结构**：
```
sys_user (用户) ←多对一→ sys_user_role (关联) ←多对一→ sys_role (角色)
                                                         ↕ 多对多
```

| '1' | 系统管理员 | 所有启用菜单 | 直接通过 |

### 九、业务模块关键实现

#### 9.1 ser-app（标准参照模块）

**分层架构**：
```
handler.go(纯分发) → ser/(业务逻辑) → rep/(数据访问) → gorm_gen/query(类型安全查询)
                                    ↗ cache/(Redis缓存)
                                    ↗ enum/(枚举注册)
                                    ↗ errs/(错误码)
```

**Banner缓存策略**：
- Key格式：`banner:list:{pagePosition}:{terminal}`，最多21个key(7位置×3终端)
- 防穿透：空列表也缓存（`[]`），区分miss(`nil`)和空结果
- 写操作后DelAll清空全部缓存key
- C端读时过滤：上线时间+语言匹配（优先请求语言→defaultLang→全部）

**Banner排序冲突处理**（事务）：
- ShiftAndCreate：同页面位置下 sort_value≥新值的记录全部+1，然后插入新记录
- 查询排序：status DESC → sort_value ASC → id DESC

**操作日志**：
- 通过 `rpc.BLogClient().AddActLog()` 异步记录
- 包含：Backend/Model/Page/Action/IsSuccess
- 编辑操作记录字段级变更对比（old→new）
- defer确保失败也记录

**枚举注册机制**（`enum_registry.go`）：
- `EnumsOptionsRegistry` map注册枚举查询函数
- 查询优先走Redis缓存→miss调注册函数→写入缓存
- 国际化枚举通过i18n key映射翻译

**URL处理**：

- 返回B端：拼接后台CDN域名（`mys3.JoinUrls(GateTypeBack,...)`）
- 返回C端：拼接前台CDN域名（`mys3.JoinUrls(GateTypeFont,...)`）

#### 9.2 ser-buser（RBAC权限）
- 四表：sys_user / sys_user_role / sys_role / sys_role_menu / sys_menu
- 登录流程：IP白名单→密码校验(salt+hash)→Google TOTP(可选)→Token(32位随机)→Redis(30天)→操作日志
- 菜单树：递归构建parent_id树，按sort字段排序
- 权限校验：超管直通，普通角色查role_menu+menu的path匹配
- 用户创建事务：INSERT user + INSERT user_role
- 角色更新事务：UPDATE role + DELETE old role_menus + INSERT new role_menus
- 错误码：6002xxx

#### 9.3 ser-bcom（B端公共）
- **开关配置**：存储在ETCD（`/slg/sett/...`），JSON格式SwitchConfigItem，13个可配开关（9注册+2访问控制+2验证）
- **黑白名单**：数据在ser-ip数据库，通过RPC操作；查询时补充操作人用户名（调BUser RPC）
- **枚举查询**：query_enum表存储enum_key/enum_value(逗号分隔)/enum_label(i18n key逗号分隔)
- **CSV导出**：bytes.Buffer + csv.Writer，返回CSVResponse接口（GetFilename+GetData）
- **邮件验证**：TODO状态（panic("implement me")），设计思路已注释（Gmail SMTP + Redis验证码5分钟TTL）
- 错误码：6003xxx

#### 9.4 ser-i18n（国际化）
- **6语言**：zh-CN/zh-TW/en/id/vi/th
- **Redis存储**：6个Hash表 `i18n:hs:{lang}`，field=翻译key(int64字符串)，value=翻译文本
- **回退链**：用户语言 → 简体中文 → 英文
- **批量导入**：每批100条，Pipeline写入6个Hash
- **Category 6类**：客户端业务/客户端报错/后台业务/后台报错/服务端业务/服务端报错
- **导出**：CSV(UTF-8 BOM)/JSON/XML三种格式，分页加载每页1000条

#### 9.5 ser-ip（IP管理）
- **黑名单**（c_black_list）：国家级，ISO 3166-1 Alpha-2 Code
- **白名单**（sys_white_list）：IP级
- **校验流程**：先查ETCD开关是否启用→IP解析国家(third.IpAnalysis)→查表
- **与网关联动**：WhiteCheck中间件调OnWhiteList，BlackCheck调OnBlackList

#### 9.6 ser-item（道具管理）
- **双表设计**：Item(基础配置,自动编号D3001+) ↔ ItemPublish(发布配置,按场景)，一对多
- **DDD风格domain包**：ItemValidator/ItemMapper/ItemCacheManager/ItemLogBuilder/UserService
- **创建事务**：INSERT Item → 生成ItemCode → 为每个ItemType创建ItemPublish（默认下架/不展示/热门标签）
- **更新事务**：UPDATE Item + 同步ItemPublish（新增类型→创建，删除类型→软删除+下架）
- **缓存**：Redis按类型和ID批量缓存，Lua脚本扫描清除`item:list:*`前缀

#### 9.7 ser-s3（文件存储）
- **直传**（≤10MB）：后端收到文件→Magic Byte检测→PutObject到S3→记录Document表
- **预签名**：GetUploadPreSign生成PUT预签名URL→客户端直传S3→UploadComplete校验MIME
- **Magic Byte检测**：GIF(GIF87a/89a) / PNG(89504E47) / JPEG(FFD8FF) / MP4(ftyp) / PAG(PAG头) / JSON({或[)
- **s3_config表**：PathTemplate(支持`{{owner_id}}/{{year}}/{{month}}/{{day}}`变量) / AllowedExt / MaxSize / IsPublic / Expires
- **配置加载**：全量读s3_config→存Redis Hash `s3:hs:config`

### 十、数据库规范（全局）

- **软删除**：`delete_at` 毫秒时间戳（0=未删除），部分表有 `delete_by`
- **审计字段**：`create_at/update_at`（毫秒时间戳）+ `create_by/update_by`（用户ID）+ `create_by_name/update_by_name`（部分表）
- **UserPlugin自动填充**：GORM插件通过反射自动设置审计字段（从tracer.GetUserId获取操作人）
- **AES字段加密**：`EncryptedString`类型实现driver.Valuer/sql.Scanner，写入自动加密读出自动解密
- **初始化模式**：`cfg.InitMySQL()` → ETCD取TiDB地址+模块DB名 → `mysql.Init(opt)` → `query.SetDefault(db)`
